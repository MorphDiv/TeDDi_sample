Support de plate-forme Java
$[officename] supporte la plate-forme Java pour exécuter des applications et des composants basés sur l'architecture JavaBeans.
Pour que $[officename] prenne en charge la plate-forme Java, vous devez installer le logiciel d'environnement d'exécution Java 2 de Sun Microsystems.
Lors de l'installation de $[officename], une option permettant d'installer ces fichiers (s'ils ne sont pas encore installés) a été automatiquement activée.
Vous pouvez également installer ces fichiers maintenant, si nécessaire.
Le fichier exécutable d'installation se trouve dans le dossier $[officename]/program et sur le CD de $[officename].
Pour que des applications Java puissent être exécutées, vous devez activer le support de plate-forme Java sous $[officename].
Outils - Options - $[officename] - Java
.
Pour utiliser un pilote JDBC, vous devez au préalable indiquer son chemin.
Choisissez Outils - Options - %PRODUCTNAME - Java, puis cliquez sur le bouton Chemin de classe.
Une fois le chemin spécifié, redémarrez %PRODUCTNAME.
Outils - Options - $[officename] - Java
sont utilisées même si une machine virtuelle Java (JVM, Java Virtual Machine) a déjà été démarrée.
Après avoir modifié la variable ClassPath, vous devez redémarrer $[officename].
Ceci vaut également pour les modifications effectuées sous
Outils - Options - Internet - Proxy
.
Seules les zones &quot; Proxy HTTP &quot; et &quot; Proxy FTP &quot; et les ports qui leur sont associés ne nécessitent pas de redémarrage.
Elles sont évaluées lorsque vous cliquez sur
OK
.
