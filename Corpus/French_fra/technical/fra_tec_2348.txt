Où chercher
Les programmes ht ://dig
htdig et htmerge sont souvent localisés dans /usr/bin/ ou dans quelque chose comme /usr/local/www/htdig/bin/.
La commande htsearch est souvent localisée dans le sous-dossier appelé cgi-bin, par exemple /usr/local/httpd/cgi-bin/.
Pour trouver où par exemple htdig est installé vous pouvez toujours saisir whereis htdig dans une console. whereis recherchera la commande spécifiée dans l'arborescence standard des dossiers.
Toutefois, les dossiers comme cgi-bin ne sont souvent pas dans cette arborescence.
Chemins de recherche et d' analyse
Dans cette section vous pouvez choisir quelles ressources d'aide doivent être indexées, i.e. rendues disponibles pour le moteur de recherche.
Dans la fenêtre Analyse vous pouvez sélectionner les ressources typiques que vous souhaitez indexer, i.e. les fichiers d'aide de KDE, et les informations offertes par les commandes man et info.
Notez que quelques unes peuvent rester indisponibles, ce qui veut dire que le support pour celles-ci n'a pas encore été ajouté.
Il est possible que vous ayez des fichiers auxquels vous voulez accéder par le moteur de recherche du centre d'aide de KDE.
Par exemple, vous pouvez avoir un fichier de référence HTML installé dans /home/jdoe/docs/selfhtml.
En ajoutant ce chemin dans la liste des chemins supplémentaires vous rendez cette documentation disponible par le moteur de recherche du centre d'aide de KDE.
Cliquez simplement sur le bouton Ajouter et une boîte de dialogue vous demandera le dossier supplémentaire cherché.
Sélectionnez /home/jdoe/docs/selfhtml et cliquez sur OK.
Pour enlever un chemin ajouté, sélectionnez le et cliquez sur Supprimer.
Les changements effectués dans l'analyse et les chemins supplémentaires n'auront pas d'effet si vous ne cliquez pas sur le bouton générer l'index....
