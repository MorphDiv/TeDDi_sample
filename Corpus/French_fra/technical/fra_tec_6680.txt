À propos de library.gnome.org
library.gnome.org a pour objectif d'être le lieu centralisant toute documentation liée au projet GNOME, qu'elle concerne les utilisateurs, les administrateurs système ou les développeurs.
Son arborescence de développement est hébergée dans le dépôt Git de GNOME, dans le module library-web.
Voir ce site web dans d'autres langues
Lorsque vous demandez un document à un serveur, votre préférence de langue est envoyée via l'en-tête HTTP Accept-Language. Il s'agit de la façon standard de fournir la version la plus appropriée d'une page disponible dans plusieurs langues. Il est important de configurer correctement la préférence de langue de votre navigateur.
Que faire si la page est affichée dans une mauvaise langue ?
La raison habituelle est que votre navigateur web n'est pas configuré correctement. Pour plus d'informations, passez à la section
Si votre navigateur est correctement configuré, le problème peut être que votre fournisseur d'accès ou votre réseau interne utilise un cache ou un serveur proxy mal configuré. Il s'agit de serveurs sans aucun contenu qui se placent entre les utilisateurs et les vrais serveurs web. Ils interceptent vos requêtes pour les pages web et téléchargent la page. Ils vous font ensuite suivre la page et créent une copie locale en cache pour de prochaines requêtes. Cela permet de diminuer le trafic réseau lorsque plusieurs utilisateurs accèdent à la même page.
Le problème est qu'ils ne comprennent pas la négociation de contenu et gardent une copie de la page en cache dans une seule langue. La seule façon de corriger cela est de vous plaindre à votre fournisseur d'accès pour qu'ils mettent à jour ou remplacent leur logiciel.
Un troisième problème peut être que vous ne pouvez pas ou ne voulez pas changer les paramètres de votre navigateur. Si JavaScript et les cookies sont activés, notre serveur sauvegardera votre préférence de langue dans un cookie à chaque fois que vous changerez de langue en cliquant sur un lien. La prochaine fois que vous cliquerez sur un lien vers une autre page, votre sélection de langue sera conservée.
Comment configurer les paramètres de langue ?
Généralement, il est préférable d'inclure dans les langues préférées toutes les langues que vous savez lire, par ordre de préférence. Il est aussi conseillé d'inclure l'anglais (« en ») comme dernier élément, comme langue à utiliser en dernier recours. Il faut être prudent avec les sous-catégories de langues : « fr-FR » n'inclut pas « fr » ; il faut donc inclure à la fois « fr » et « fr-FR » dans votre liste.

Demande d'ajout d'une documentation
Library.gnome.org gère les formats gnome-doc-utils (manuels) et gtk-doc (références). Il sait traiter de façon minimale les fichiers HTML au travers de feuilles de style XSL. Il peut aussi afficher des liens vers des ressources disponibles sur d'autres sites Web.

Il faut relever que le site library.gnome.org ne traite que des archives (tarballs) et qu'il ne gère pas les documents qui ne sont disponibles que dans un système de gestion de versions. La première chose à faire avant de demander l'inclusion d'un nouveau document est de s'assurer qu'il existe sous forme d'archive. Ce site sait aussi gérer spécialement les archives hébergées sur download.gnome.org et il se charge de récupérer automatiquement les nouvelles versions.
