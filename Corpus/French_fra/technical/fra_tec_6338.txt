Format de l'heure
Cette clé indique le format d'heure utilisé par l'applet horloge. Les valeurs possibles sont « 12-hour » (12 heures), « 24-hour » (24 heures), « internet » (Internet), « unix » (Unix) et « custom » (personnalisé). Si elle est définie à « internet », l'horloge affiche le temps Internet. Le système de temps Internet divise le jour en 1 000 segments. Il n'y a pas de fuseaux horaires dans ce système, donc le temps est le même partout dans le monde. Si elle est définie à « unix », l'horloge affiche le temps en secondes depuis Epoch, c'est à dire le 1er janvier 1970. Si elle est définie à « custom » (personnalisé), l'horloge affiche l'heure en fonction du format indiqué dans la clé custom_format.
Format personnalisé de l'horloge
Cette clé indique le format utilisé par l'applet horloge lorsque la clé format est définie à « custom ». Vous pouvez utiliser des spécificateurs de conversion compris par strftime() pour obtenir un format spécifique. Consultez le manuel de strftime() pour de plus amples informations.

Nom d'identification du tableau de bord
C'est un nom humainement lisible qui identifie le tableau de bord. Son rôle principal est de servir comme titre de fenêtre du tableau de bord, ce qui est utile pour naviguer entre les tableaux de bord.
Le visuel X sur lequel le tableau de bord est affiché
Avec un réglage multi-écran, vous pouvez avoir des tableaux de bord sur chacun des écrans. Cette clé identifie l'écran sur lequel le tableau de bord est affiché.
Écran Xinerama sur lequel le tableau de bord est affiché
Dans un mode Xinerama, vous pouvez avoir des tableaux de bord sur chacun des écrans. Cette clé identifie l'écran sur lequel le tableau de bord est affiché.
Étendre pour occuper la largeur totale de l'écran
Si vrai, le tableau de bord occupe toute la largeur de l'écran (hauteur si c'est un tableau de bord vertical). Dans ce mode, le tableau de bord est placé sur le côté de l'écran. Si faux, le tableau de bord est assez large pour recevoir les applets, lanceurs et boutons du tableau de bord.
Orientation du tableau de bord
L'orientation du tableau de bord. Les quatre valeurs possibles sont : « top » (haut), « bottom » (bas), « left » (gauche) et « right » (droite). En mode étendu, la clé indique sur quel côté de l'écran placer le tableau de bord. En mode non étendu, la différence entre « top » et « bottom » est moins importante - les deux indiquent que c'est un tableau de bord horizontal - mais donne toutefois une indication utile sur la manière dont certains éléments de tableau de bord doivent se comporter. Par exemple, sur un tableau de bord « top », un bouton de menu affiche le menu sous le tableau de bord, alors qu'avec un tableau de bord « bottom » le menu est affiché au-dessus du tableau de bord.
Taille du tableau de bord
La hauteur (largeur pour un tableau de bord vertical) du tableau de bord. Le tableau de bord détermine à l'exécution une taille minimum basée sur la taille de la police et d'autres indicateurs. La taille maximum est fixée à un quart de la hauteur (ou largeur) de l'écran.
Abscisse du tableau de bord
L'emplacement du tableau de bord selon l'abscisse. Cette clé n'est valable que dans le mode non étendu. En mode étendu cette clé est ignorée et le tableau de bord est placé sur le côté de l'écran indiqué par la clé orientation.
Ordonnée du tableau de bord
L'emplacement du tableau de bord selon l'ordonnée. Cette clé n'est valable que dans le mode non étendu. En mode étendu cette clé est ignorée et le tableau de bord est placé sur le côté de l'écran indiqué par la clé orientation.
Abscisse du tableau de bord, à partir de la droite de l'écran
L'emplacement du tableau de bord selon l'abscisse (x), à partir de la droite de l'écran. Si cette valeur vaut -1, elle est ignorée et c'est la valeur de la clé x qui est utilisée. Si cette valeur est plus grande que 0, la valeur de la clé x est ignorée. Cette clé n'est valable que dans le mode non étendu. En mode étendu, cette clé est ignorée et le tableau de bord est placé sur le côté de l'écran indiqué par la clé orientation.
Ordonnée du tableau de bord, à partir du bas de l'écran
L'emplacement du tableau de bord selon l'ordonnée (y), à partir du bas de l'écran. Si cette valeur vaut -1, elle est ignorée et c'est la valeur de la clé y qui est utilisée. Si cette valeur est plus grande que 0, la valeur de la clé y est ignorée. Cette clé n'est valable que dans le mode non étendu. En mode étendu, cette clé est ignorée et le tableau de bord est placé sur le côté de l'écran indiqué par la clé orientation.
Centrer le tableau de bord sur l'axe X
Si vrai, les clés x et x_right sont ignorées et le tableau de bord est placé au centre de l'axe X de l'écran. Si le tableau de bord est redimensionné, il reste à cette position, c'est-à-dire qu'il grandit des deux côtés. Si faux, les clés x et x_right déterminent l'emplacement du tableau de bord.
Centre le tableau de bord sur l'axe Y
Si vrai, les clés y et y_bottom sont ignorées et le tableau de bord est placé au centre de l'axe Y de l'écran. Si le tableau de bord est redimensionné, il reste à cette position, c'est-à-dire qu'il grandit des deux côtés. Si faux, les clés y et y_bottom déterminent l'emplacement du tableau de bord.
Masquer automatiquement le tableau de bord dans le coin
Si vrai, le tableau de bord est automatiquement caché dans un coin de l'écran lorsque le pointeur quitte l'aire du tableau de bord. Déplacer le pointeur vers ce coin fait réapparaître le tableau de bord.
Si vrai, le masquage et l'affichage de ce tableau de bord sont animés au lieu d'arriver instantanément.
Activer les boutons de masquage
Si vrai, des boutons sont placés de chaque côté du tableau de bord et peuvent être utilisés pour déplacer le tableau de bord sur les côtés de l'écran, laissant seulement un bouton d'affichage.
Activer les flèches sur les boutons de masquage
Si vrai, les flèches sont placées sur les boutons de masquage. Cette clé n'est valable que si la clé enable_buttons est active.
Délai de masquage automatique du tableau de bord
Indique le nombre de millisecondes du délai entre le moment où le pointeur quitte le tableau de bord et le moment où le tableau de bord est automatiquement caché. Cette clé n'est valable que si la clé auto_hide est à vrai.
Délai d'affichage automatique du tableau de bord
Indique le nombre de millisecondes du délai entre le moment où le pointeur entre dans l'aire du tableau de bord et le moment où le tableau de bord est réaffiché. Cette clé n'est valable que si la clé auto_hide est à vrai.
Nombre de pixels visibles lorsque caché
Indique le nombre de pixels visible lorsque le tableau de bord est automatiquement caché dans un coin. Cette clé n'est valable que si la clé auto_hide est à vrai.
Vitesse de l'animation
La vitesse à laquelle les animations doivent fonctionner. Il y a 3 valeurs possibles : « slow » (lent), « medium » (moyen) et « fast » (rapide). Cette clé n'est valable que si la clé enable_animations est à vrai.
Type d'arrière-plan
Le type d'arrière-plan utilisé avec ce tableau de bord. Les valeurs possibles sont : « gtk », où l'arrière-plan GTK+ par défaut est utilisé, « color » où la clé color est utilisée comme couleur d'arrière-plan ou « image » où l'image indiquée par la clé image est utilisée comme arrière-plan.
Couleur de l'arrière-plan
Indique la couleur de l'arrière-plan des tableaux de bord au format #RGB.
Opacité de la couleur de l'arrière-plan
Indique le format de l'opacité de la couleur d'arrière-plan. Si la couleur n'est pas complètement opaque (une valeur de moins de 65535), la couleur est composée avec l'image d'arrière-plan du bureau.
Image d'arrière-plan
Indique le fichier à utiliser comme image d'arrière-plan. Si l'image contient un canal alpha, elle est composée avec l'image d'arrière-plan du bureau.
Ajuster l'image au tableau de bord
Si vrai, l'image est mise à l'échelle (en conservant le ratio d'aspect de l'image) à la hauteur du tableau de bord (si horizontal).
Étire l'image dans le tableau de bord
Si vrai, l'image est redimensionnée aux dimensions du tableau de bord. Le ratio d'aspect de l'image n'est pas maintenu.
Pivoter l'image dans les tableaux de bord verticaux
Si vrai, l'image d'arrière-plan est pivotée lorsque le tableau de bord est orienté verticalement.
