Général
Restrictions générales
Désactiver le menu contextuel du gestionnaire de fenêtres (Alt-F3)
Le menu contextuel du gestionnaire de fenêtres est normalement affiché lorsqu'on appuie sur Alt-F3 ou sur le bouton de menu situé sur le cadre de la fenêtre.
Désactiver les signets
Désactive les signets dans toutes les applications.
Désactiver toutes les tâches et applications nécessitant un accès de superutilisateur
Dans les environnements multi-utilisateurs, les utilisateurs normalement ne connaissent pas le mot de passe superutilisateur (root). Dans un tel cas, il peut être souhaitable d'utiliser cette option ou de supprimer les tâches et applications depuis les menus que les utilisateurs ne peuvent employer.
Désactiver l'accès à un interpréteur de commandes
Dans un environnement où le bureau est dédié à un ensemble défini de tâches, il peut être souhaitable de désactiver l'accès à un interpréteur de commandes pour empêcher les utilisateurs d'entreprendre des tâches qui n'ont pas été planifiées ou autorisées. Par conséquent, il est vivement recommandé de désactiver l'accès à un interpréteur de commandes si le bureau doit agir en tant que terminal public.
Désactiver l'option Déconnexion
Cette option empêche l'utilisateur de se déconnecter. Pour qu'elle entre en vigueur, il est important de veiller à ce que les combinaisons de touches pour fermer le serveur & #160; X, telles que Alt-Ctrl-Correction, soient désactivées dans la configuration du serveur & #160; X.
Désactiver l'option Verrouillage de l' écran
Lorsque le bureau doit agir en tant que terminal public ou être partagé par différents utilisateurs, il peut être souhaitable d'empêcher le verrouillage de l'écran pour garantir que le système reste accessible si un utilisateur quitte le terminal.
Désactiver l'option Exécuter une commande (Alt-F2)
L'option Exécuter une commande peut normalement être utilisée pour exécuter des commandes arbitraires. Cependant, quand l'accès à un interpréteur de commandes est restreint, seuls les applications et les services définis par un fichier. desktop peuvent être démarrés de cette façon. Désactiver l'option Exécuter une commande ici masque l'option complètement.
Désactiver le déplacement des barres d' outils
Normalement, dans les applications, les barres d'outils peuvent se déplacer. Lorsque cette option est cochée, toutes les barres d'outils sont fixes dans leurs positions d'origine.
Désactiver l'exécution de fichiers. desktop arbitraires
Cette option détermine si les utilisateurs peuvent ou non exécuter les fichiers. desktop qui ne font pas partie des icônes de bureau à l'échelle du système, le menu KDE, les services enregistrés ou les services à lancement automatique. Lorsque l'accès à un interpréteur de commandes est restreint, il est recommandé de désactiver l'exécution des fichiers. desktop arbitraires également parce que de tels fichiers. desktop peuvent être utilisés pour contourner la restriction sur l'interpréteur de commandes.
Désactiver le démarrage d'une seconde session de X
KDM a la possibilité de connecter un second utilisateur en parallèle à la session en cours. Notez que celle -ci peut également être configurée comme composant des paramètres de KDM, auquel cas le réglage devra ici rester activé.
Désactiver l'historique de la ligne de commande
Si un seul compte est utilisé par plusieurs personnes, il peut être souhaitable de désactiver l'historique de la ligne de commande pour des considérations relatives à la vie privée.
Appliquer les restrictions de DRM
Certaines applications KDE peuvent appliquer des restrictions de DRM sur le contenu. Vous pouvez être amené à cocher cette option si votre organisation repose sur de telles restrictions ou si la législation locale l'exige.
Empêcher le téléchargement
Certaines applications KDE peuvent empêcher l'utilisateur de télécharger ou envoyer des fichiers. La pratique normale de la navigation web n'en sera pas affectée. Notez qu'il vous faut encore régler les permissions sur les fichiers Unix correctement - considérez cette option comme étant purement cosmétique pour la sécurité.
Icônes du bureau
Les icônes du bureau sont fournies par « & #160; kdesktop & #160; ».
Verrouiller les paramètres du bureau
Lorsque les paramètres du bureau sont verrouillés, l'utilisateur ne peut plus modifier le comportement ou l'apparence du bureau. Cette option n'affecte pas la capacité à ajouter de nouveaux fichiers ou raccourcis au bureau.
Désactiver les menus contextuels
Lorsque cette option est cochée, l'utilisateur ne peut plus obtenir de menu contextuel. Normalement, l'utilisateur peut faire apparaître un menu contextuel en cliquant avec le bouton droit de la souris.
Verrouiller toutes les icônes du bureau
Lorsque cette option est cochée, l'utilisateur n'est plus en mesure de supprimer ou de modifier une icône ou un fichier existant sur le bureau, ni ajouter de nouvelles icônes ou de nouveaux fichiers.
Verrouiller les icônes du bureau à l'échelle du système
Lorsque cette option est cochée, l'utilisateur n'est plus en mesure de supprimer ou de modifier aucune des icônes à l'échelle du système mais pourra toujours ajouter, supprimer ou modifier des icônes ou des fichiers personnels sur le bureau.
Ajouter des icônes supplémentaires spécifiques à un fournisseur
Lorsque cette option est cochée, les utilisateur verront leurs icônes supplémentaires copiées sur le bureau lors de leur première connexion. Ces icônes n'apparaissent pas dans l'aperçu.

Certains écrans de veille ne masquent pas le contenu de l'écran complet et peuvent laisser visibles des informations potentiellement sensibles. Cette option désactive tous les écrans de veille de ce type et n'active que ceux qui masquent complètement le contenu original de l'écran.
