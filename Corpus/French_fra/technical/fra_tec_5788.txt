Gestionnaire de démarrage automatique
Ce module est un outil de configuration pour gérer quels programmes démarrer avec & kde;. Il vous permet d'ajouter des programmes et des scripts de façon à les démarrer automatiquement au lancement de & kde; et de les gérer.
Remarquez que dans ce module, toutes les modifications prennent effet immédiatement.
Le programme analyse ~/. config/ autostart/, votre dossier Démarrage automatique de KDE ($KDEHOME/ Autostart par défaut), les dossiers $KDEHOME/ env et $KDEHOME/ shutdown pour vérifier quels programmes et scripts sont déjà présents et les affiche. Il vous permet de les gérer facilement.
Remarquez que vous pouvez modifier l'emplacement du dossier Autostart dans l'onglet À propos de moi dans la & configurationDuSysteme; et choisir un dossier autre que $KDEHOME/ Autostart.
Affichage des fichiers
La partie principale du module affiche les programmes qui sont chargés et les scripts qui sont exécutés lorsque & kde; démarre, se ferme, ou avant que & kde; démarre.
Nom
Vous en pouvez pas modifier cette colonne. Elle donne le nom du programme ou du script que vous voulez démarrer avec & kde;. Le nom est extrait du fichier Desktop depuis la clé Nom pour un programme. Pour un script, il s'agit du nom du fichier.
Commande
Cette colonne liste les commandes qui sont exécutées pour démarrer le programme. Les commandes peuvent être modifiées avec le bouton Propriétés ou en double cliquant sur la ligne du programme ou du script. La commande est extraite du fichier Desktop depuis la clé key.
Pour un script, la commande est l'emplacement vers le script.
État
Ce réglage n'est disponible que pour les programmes à travers les fichiers Desktop. Vous pouvez conserver un programme dans le dossier Autostart mais le désactiver au démarrage de & kde;. Régler le programme sur Désactivé ne lancera pas le programme au démarrage.
Régler un programme sur Désactivé rend le fichier Desktop caché dans le dossier Autostart.
S'exécute sur
Les programmes (fichiers Desktop) ne peuvent être exécutés qu'au démarrage. Les scripts peuvent être exécutés au démarrage, à la fermeture ou juste avant KDE. Cette colonne vous permet de modifier le moment du lancement du script. Au démarrage signifie lorsque KDE démarre, à la fermeture correspond au moment où vous vous déconnectez de & kde; et juste avant KDE signifie juste avant que & kde; ne démarre.
Les scripts et fichiers Desktop réglés pour s'exécuter au démarrage sont copiés ou liés par un lien symbolique dans $KDEHOME/ Autostart et seront exécutés pendant le démarrage de & kde;.
Les scripts configurés pour être exécutés à la fermeture sont copiés ou liés par un lien symbolique dans le dossier $KDEHOME/ shutdown et seront exécutés automatiquement pendant la fermeture de & kde; après que l'utilisateur s'est déconnecté.
Les scripts configurés pour s'exécuter juste avant & kde; sont copiés ou liés par un lien symbolique dans $KDEHOME/ env et sont « & #160; sourcés & #160; » au démarrage de & kde; (le script startkde cherchera les scripts ici).
Seuls les scripts avec l'extension sh peuvent être lus par & kde; pour les modes de démarrage juste avant KDE et à la fermeture.
Actions
Sur la droite, vous avez quelques boutons pour modifier la configuration du démarrage automatique. Vous pouvez ajouter des programmes ou des scripts, les enlever ou modifier leurs propriétés.
Ajouter un programme
Cliquer sur ce bouton affiche la boîte de dialogue standard de & kde; Choisir une application et vous permet de choisir quel programme vous souhaitez démarrer. Après avoir choisi le programme, cliquez sur Ok et les propriétés du programme s'afficheront.
Ceci copiera le fichier Desktop du programme dans votre dossier Autostart.
Ajouter un script
Ce bouton ouvre une boîte de dialogue vous demandant l'emplacement du script que vous souhaitez ajouter. Si vous laissez cocher Créer en tant que lien symbolique (par défaut), alors le script sera ajouté en tant que lien symbolique. Si vous décochez cette option, alors le script sera copié dans le dossier local correspondant.
Enlever
Enlèvera immédiatement le fichier Desktop pour les programmes, le script ou le lien symbolique du dossier Autostart.
Propriétés
Ce bouton vous permet de modifier les propriétés du programme ou du script. Vous avez les propriétés générales, les droits d'accès, un aperçu le cas échéant et des propriétés liées à l'application dans le cas des programmes.
Expert
Ce bouton ne concerne que les programmes (cad; les fichiers Desktop). Une boîte de dialogue vous demande si vous souhaitez que les programmes soient démarrés uniquement dans & kde; (et pas dans les autres environnements de bureau). Par défaut, le programme démarrera automatiquement dans tous les environnements de bureau. Cochez Démarrage automatique seulement dans KDE pour que le programme ne soit démarré automatiquement que si vous démarrez l'environnement de bureau & kde;.
Ceci règle la valeur KDE pour la clé OnlyShowIn dans le fichier Desktop du programme.
