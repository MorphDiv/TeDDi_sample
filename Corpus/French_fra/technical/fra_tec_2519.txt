Manuel de KMail
KMail est un client de courrier électronique très puissant pour KDE
Introduction
L'équipe de KMail vous souhaite la bienvenue dans KMail, gestionnaire de courrier électronique du K Desktop.Environment.
Notre but a été de concevoir un logiciel à la fois beau, intuitif et riche en possibilités.
Si vous configurez pour la première fois un client de courrier électronique sur un système UNIX, lisez d'abord la section Débuter pour démarrer tranquillement.
Vous n'avez pas besoin d'utiliser votre souris pour utiliser KMail. tout peut être fait en utilisant les Raccourcis clavier.
Bien que KMail puisse être considéré comme fiable, vous devriez tout de même faire des copies de vos messages, c'est-à-dire recopier simplement les fichiers situés dans ~/Mail dans un endroit sûr (y compris les fichiers cachés, qui commencent par un point).
Débuter
Cette courte introduction à KMail et à son utilisation vous permettra de l'utiliser rapidement.
L'installation du logiciel est décrite dans l'appendice.
Lors de son premier lancement, KMail crée dans votre dossier d'accueil un dossier appelé Mail qui contient les fichiers initiaux (inbox, outbox, sent-mail, trash et drafts).
Pour commencer à envoyer et recevoir des messages, vous n'aurez qu'à changer les réglages dans les pages Identité et Réseau

Si vous ne savez pas quels paramètres choisir ou quoi saisir dans un champ, consultez votre Fournisseur d'accès à Internet (FAI) ou votre administrateur système.

Panneau des dossiers (à gauche)
Ce panneau contient la liste de vos dossiers (les autres clients de messagerie les appellent généralement boîtes aux lettres).
S'il y a un nombre après le nom du dossier, ceci indique qu'il contient des messages non lus, sauf pour la boîte À envoyer qui ne montre que le nombre total de messages.
Pour sélectionner un dossier, cliquez simplement dessus.
Les messages contenus dans le dossier apparaîtront dans le panneau des en-têtes.
Le nombre de messages non lus et le nombre total de messages dans le dossier sélectionné sont maintenant affichés dans la barre d'état.
Les dossiers peuvent être affichés au format court, ce qui ne prend qu'une petite portion du coté gauche de l'écran, et au format long, qui prend tout le coté gauche de l'écran, mais qui peut afficher plus de boîtes aux lettres.
Voir aussi la section dossiers
Panneau des en-têtes (panneau en haut à droite)
Ce panneau liste les informations sur les en-têtes des messages du dossier actuellement sélectionné (état du message, expéditeur, sujet, date et optionnellement taille).
Si vous cliquez sur un en-tête, le message sera sélectionné et affiché dans le panneau des messages.
Vous pouvez également sélectionner une série de messages en cliquant sur un message, en maintenant enfoncée la touche Maj et en cliquant sur un autre message.
Les deux messages sur lesquels vous venez de cliquer seront sélectionnés, ainsi que tous les messages qui se trouvent entre les deux.
En maintenant enfoncée la touche Ctrl, vous pouvez sélectionner n'importe quels messages sans inclure ceux situés entre.
Vous pouvez trier les messages en cliquant sur la colonne selon laquelle vous voulez trier.
Si vous cliquez plus d'une fois sur la même colonne, l'ordre de tri sera inversé (croissant/décroissant).
Si vous cliquez sur la colonne Sujet, vous pouvez basculer entre les tris Sujet croissant/décroissant et État croissant/décroissant.
De manière similaire, si vous cliquez sur la colonne Date, vous pouvez basculer entre les tris Date croissant/décroissant et Date (ordre d'arrivée)

Vous pouvez également sélectionner une méthode d'encodage pour votre fichier dans la liste (normalement, ça devrait fonctionner avec la valeur par défaut).
Cochez l'option Suggérer un affichage automatique si vous voulez suggérer au destinataire l'affichage automatique de cette partie dans l'aperçu du message au lieu de la vue en icône par défaut.

Vous devez encrypter vos fichiers joints manuellement, en utilisant une commande shell ou avec d'autres outils avant de les joindre à votre message.

Filtres de messages
Après avoir utilisé KMail pendant un certain temps, vous vous rendrez compte qu'il est un peu difficile de trier tous les nouveaux messages dans la boîte de réception.
Les filtres vous permettent de réaliser automatiquement certaines actions sur les messages entrants et de réaliser des actions manuellement sur les messages sélectionnés dans un dossier.
Veuillez noter que les filtres décrits dans cette section sont appliqués après que les messages ont été téléchargés de votre serveur.
Si vous voulez filtrer les messages sur le serveur, voyez Filtrage des téléchargements.
Les filtres consistent en un motifs de recherche, dont les règles sont utilisées comme critère pour déterminer si le filtre doit être appliqué ou non à un certain message, et une liste d'actions, qui décrivent ce qui doit être réalisé sur le message si le motif de recherche concorde.
Si vous souhaitez en savoir plus sur les motifs de recherche et les actions des filtres, lisez les sections suivantes.
Les filtres sont appliqués les uns après les autres, en partant du premier filtre de la liste.
Le premier filtre pour lequel le message correspond au motif est exécuté.
Vous pouvez demander à ce que les filtres restants soient également appliqués, mais par défaut, le traitement s'arrête à la première concordance.
Habituellement, les filtres sont utilisés pour les messages entrants, mais ils peuvent également être appliqués à un message ou à un groupe de messages.
Ceci appliquera tous vos filtres aux messages sélectionnés pour le filtrage manuel dans la boîte de dialogue des filtres à ces messages.
Création rapide d'un filtre
Il y a deux méthodes pour créer un filtre.
Ceci fait apparaître la boîte de dialogue des règles de filtrage et met à votre disposition un nouveau filtre contenant un premier motif de recherche et une première action (comme mettre dans le dossier).
Dans la plupart des cas, tout ce que vous avez à faire est de sélectionner le dossier dans lequel vous voulez déplacer les messages.
Vous pouvez bien sûr éditer le filtre comme vous le souhaitez.
Ce qui est intéressant ici, c'est que, pour les listes de diffusion, un en-tête unique identifiant les courriers de cette liste sera recherché.
Cette boîte de dialogue est décrite en détail dans la section ci-dessous.
La boîte de dialogue des règles de filtrage
Cette boîte de dialogue vous permet de gérer et éditer la liste de vos filtres.
Les filtres disponibles
Ce groupe contient la liste des filtres et des boutons d'action pour modifier les filtres, c'est-à-dire pour créer de nouveaux filtres, les déplacer vers le haut ou vers le bas de la liste, les supprimer ou les renommer.
Si vous sélectionnez un filtre dans la liste, ses propriétés sont affichées dans la moitié droite de la boîte de dialogue.
Critères de filtrage
Dans ce groupe, vous pouvez éditer le motif auquel devront correspondre les messages afin que le filtre leur soit appliqué.
Vous pouvez sélectionner ici soit que tous les critères doivent correspondre, soit qu'au moins un des critères doit correspondre.
Consultez Motifs de recherche ci-dessous pour une description détaillée de chaque règle de recherche.
Vous pouvez cliquer sur En ajouter une autre pour obtenir une règle de filtrage supplémentaire (initialement vide), si vous désirez définir des motifs de recherche plus complexes.
Cliquez sur En supprimer une pour enlever la dernière règle.
Effacer efface le motif, c'est-à-dire qu'il enlève toutes les règles de l'écran sauf deux, et les réinitialise.
Les règles non valides ou vides ne sont pas évaluées.
Actions du filtre
Dans ce groupe, vous pouvez éditer la liste des actions qui seront appliquées à tous les messages qui correspondent au motif de recherche.
Voir Actions du filtre ci-dessous pour une description détaillée de chaque type d'action.
Vous pouvez cliquer sur En afficher une autre pour obtenir une action supplémentaire (initialement vide) si vous désirez définir plus d'une action.
Cliquez sur En supprimer une pour enlever la dernière action.
Effacer efface la liste, c'est-à-dire qu'il enlève toutes les actions de l'écran sauf une, et la réinitialise.
Les actions non valides ou vides ne sont pas exécutées.
Options avancées
Dans ce groupe, vous pouvez définir quelques options avancées pour les filtres, ce qui vous permet de peaufiner votre filtrage.
En utilisant la première ligne de cases à cocher, vous pouvez déterminer comment s'applique le filtre correspondant.
Si vous choisissez aux nouveaux messages, le filtre sera appliqué uniquement sur les messages entrants (c'est-à-dire avec Relever tout le courrier).
Les filtres sont automatiquement nommés tant que vous ne les renommez pas de manière explicite en utilisant le bouton Renommer.
Cette boîte de dialogue permet de nommer automatiquement les filtres tant que le nom du filtre commence par <..
Si vous appliquez vos changement, via OK ou Appliquer, seuls les filtres valides sont réellement copiés dans le gestionnaire de filtres interne.
Un filtre est considéré non valide si
le motif de recherche contient uniquement des règles vides ou
la liste des actions est vide.
De manière similaire, les règles et les actions vides sont enlevées du motif et de la liste des actions respectivement, avant d'enregistrer le filtre.
Motifs de recherche
<message>
Cherche dans l'intégralité du message (c'est-à-dire l'en-tête, le corps et les fichiers joints le cas échéant).
<corps>
Cherche dans le corps du message.
<n'importe quelle en-tête>
Cherche dans les en-têtes du message.
<destinataires>
Cherche dans les champs d'en-tête du message À et Cc.
<taille>
Règle les limites inférieure ou supérieure de la taille du message.
<âge en jours>
Règle les limites inférieure ou supérieure de l'âge du message.
N'importe quelle autre chaîne
Cherche dans le champ d'en-tête donné par cette chaîne.
Action du filtre
mettre dans le dossier
déplace le message dans un autre dossier.
Vous ne pouvez pas actuellement déplacer dans des dossier IMAP.
définir l'identité comme étant
détermine votre identité si vous répondez à ce message.
marqué comme
marque le message comme lu ou important (drapeau), mais aussi comme transmis, répondu, etc..
définir le transport comme
définit la méthode de transport (par exemple SMTP) si vous répondez au message.
fixe l'adresse de réponse comme
modifie le champ Répondre à pour ce message.
Ceci peut être utile pour les listes de discussion qui règlent automatiquement l'adresse de réponse à une adresse qui ne vous plaît pas.
transmettre à
transmet le message à une autre adresse électronique.
rediriger
redirige le message vers une autre adresse électronique.
refuser
retourne le message à l'expéditeur comme étant non délivrable.
Ceci ne fonctionnera que si l'adresse électronique de l'expéditeur est valide.
Ce n'est pas la cas la plupart du temps lorsqu'il s'agit de spam.
confirmer la réception
essaie de renvoyer un message à l'expéditeur lui indiquant que son message a bien été reçu.
Cette action vous permet de sélectionner qui recevra des accusés de réception de votre part.
exécuter la commande
utiliser le programme de filtre
Cela chargera le message dans un programme.
Si ce programme génère une sortie, le message entier (y compris ses en-têtes) sera remplacé par cette sortie.
Si ce programme ne génère pas de sortie, le message restera inchangé.
Vous devez donner le chemin complet vers le programme.
Soyez prudent avec cette action, elle peut facilement détériorer vos messages si le programme de filtre vous retourne des déchets ou des lignes supplémentaires.
supprimer l' en-tête
supprime tous les champs d'en-tête du message avec un nom donné.
Ceci est utile principalement pour enlever les en-têtes Répondre à boguées.
ajouter l' en-tête
récrire l' en-tête
Va balayer le champ de l'en-tête donné, modifier son contenu puis l'enregistrer ainsi.
La chaîne de caractères à rechercher est toujours interprétée comme ayant une casse régulière.
La chaîne de remplacement est insérée littéralement, sauf pour des occurrences de \n, $n et ${nn}, où n est un nombre positif (nombre à un seul chiffre, sauf pour la troisième forme) ou 0.
Entourées de parenthèses au sein de la chaîne de recherche, ces constructions sont interprétées comme des renvois aux sous-chaînes.
Des restrictions analogues à celles appliquées lors de l'action ajouter un en-tête s'appliquent aussi ici.

Filtres de téléchargement
Les filtres de téléchargement peuvent être utilisés pour filtrer les courriers d'un serveur POP avant qu'ils soient téléchargés.
Vous pouvez utiliser ceci pour empêcher KMail de télécharger d'énormes courriers et ainsi économiser du temps.
Faites attention à l'option Supprimer les messages sur le serveur car une fois un message supprimé sur le serveur, il n'y a plus moyen de le récupérer.
La boîte de dialogue Configuration des filtres POP
Télécharger le message
Téléchargera les messages correspondant au filtre, exactement comme n'importe quel autre message qui n'excède pas le seuil de taille.
Télécharger le message plus tard
Marquera les messages pour un téléchargement ultérieur.
Ceci signifie que les messages correspondants resteront sur le serveur POP jusqu'à ce que vous décidiez de les télécharger en changeant l'action à la main.
Supprimer le message du serveur
Supprimera le message du serveur et ne le téléchargera pas.
Une fois que vous avez supprimé un message du serveur, il n'y a pas de manière de le récupérer.
Faites attention, car les règles pourraient concerner des messages que vous voudriez en fait.
La boîte de dialogue de confirmation
Cette boîte de dialogue s'affiche au cas où vous auriez un filtrage POP activé et que des messages excédant le seuil que vous avez défini pour le compte POP ont été trouvés sur le serveur.
Maintenant, vous avez l'opportunité de décider ce que vous voulez faire de ce message.
Les options sont télécharger (vert), télécharger plus tard (jaune avec une montre) et Supprimer du serveur (x rouge).
Faites attention avec l'option de suppression, car une fois supprimés sur le serveur, il n'y a pas moyen de les récupérer.
Dans la section Messages filtrés, vous pouvez cocher la case si vous avez des messages qui ont été automatiquement marqués pour une certaine action (télécharger, télécharger plus tard, supprimer) par une règle de filtrage.
La case à cocher n'est activée que si vous avez des messages qui correspondaient à une règle de filtration.
Une fois que vous l'avez cochée, une liste semblable à celle pour les messages non marqués automatiquement s'affichera et vous pourrez changer l'action pour chaque message.
Veuillez noter que s'il y a un message qui dépasse la taille limite, mais que tous les messages sont marqués par une règle de filtrage, le dialogue ne s'affichera pas.
Une exception survient si vous avez coché Toujours afficher les message «à télécharger plus tard» dans la fenêtre de confirmation dans la section Options globales de la boîte de dialogue de configuration des filtres POP.
Alors, le dialogue s'affichera aussi si vous avez seulement des messages marqués, mais au moins un a été marquée pour Télécharger plus tard.
Utilisation de comptes multiples
Les comptes multiples sont utilisés pour relever les messages de plusieurs adresses électroniques et/ou de plusieurs serveurs.
Référez -vous à la section Démarrer pour avoir plus d'informations sur la configuration de la page Réseau.
Vous pouvez également cliquer en exerçant une pression prolongée sur le bouton Relevez le courrier dans pour obtenir une liste des comptes.
Signer et crypter les messages avec PGP ou GnuPG
Vous pouvez encrypter vos messages envoyés avec KMail, mais cela ne signifie pas que les fichiers joints seront encryptés.
Vous devez encrypter vos fichiers joints manuellement, en utilisant une commande shell ou avec d'autres outils avant de les joindre à votre message.
KMail dépend de la sortie de PGP.
Cette sortie est souvent différente selon les versions de PGP.
Il est donc important que vous testiez le cryptage pour savoir s'il fonctionne réellement avec votre configuration avant de l'utiliser sérieusement.
KMail peut ne pas vous prévenir si quelque chose ne fonctionne pas correctement.
Vous avez donc intérêt à activer l'option Afficher le texte crypté/signé après l'édition.
C'est certainement une bonne idée d'étudier cette documentation ainsi qu'une introduction à la cryptographie par clé publique (par exemple dans le paquetage PGP 6.5.x).
Là, vous pourrez en apprendre beaucoup sur les concepts de base, qui vous aideront à comprendre ce qui se passe.
De plus, beaucoup de problèmes liés à la sécurité que vous devriez connaître sont débattus ici.
Prérequis
KMail s'attend à ce que votre binaire PGP soit nommé pgp.
En cas de GnuPG, il s'attend à un binaire nommé gpg.
Si ce n'est pas le cas, faites simplement un lien symbolique.
Réglages en relation avec PGP dans KMail.
Outil de cryptage
Ici, vous pouvez choisir si vous voulez utiliser PGP, GnuPG ou aucun logiciel de cryptographie.
Bien sûr, il faut que le programme sélectionné soit installé sur votre système.
Il faut également faire attention à choisir la bonne version.
Conserver la phrase-clef en mémoire
Quand cette option est désactivée, KMail vous demandera la phrase de passe à chaque fois que vous signerez un message (avant envoi) ou sélectionnerez un message crypté.
Si vous activez cette option, KMail se souviendra de votre phrase de passe après votre première saisie réussie jusqu'à ce que vous ayez fini votre session de KMail.
La phrase de passe est enregistrée en mémoire et pas sur le disque dur.
Crypter les messages envoyés à soi -même
Si cette option est désactivée et que vous voulez envoyer un message crypté à quelqu'un, vous ne pourrez plus le lire après l'avoir composé et crypté.
Activez cette option pour conserver les messages cryptés lisibles par vous-même.
Afficher le texte crypté/signé après l' édition
Cela vous présentera le résultat du cryptage ainsi que de la signature avant que le message ne soit envoyé.
Ainsi, vous pourrez toujours annuler l'envoi si le cryptage a échoué.
Il vous est chaudement recommandé d'utiliser cette option.
Toujours montrer la clé de cryptage pour approbation
Ceci vous permet d'ouvrir une boîte de dialogue qui vous permet de choisir la clé utilisée pour chaque destinataire à qui vous envoyez un message crypté.
Si cette option est désactivée, KMail vous proposera toujours ce dialogue quand il ne trouvera pas de clé pour un destinataire.
Désormais, vous pouvez signer les messages que vous envoyez.
Pour que d'autres personnes puissent vous envoyer des messages cryptés et pour qu'ils puissent vérifier votre signature, vous devez leur envoyer votre clef publique ou la publier sur un serveur de clefs publiques PGP sur lequel ils pourront la récupérer.
Pour envoyer des messages cryptés à d'autres personnes ou pour vérifier leurs messages signés, vous allez avoir besoin de leurs clefs publiques.
Vous pouvez stocker votre clef publique sur un serveur de clefs publiques PGP tel que http ://www.cam.ac.uk.pgp.net/pgpnet/.
Signer vos messages
Vous composez votre message dans la fenêtre habituelle de l'éditeur de KMail.
Avant d'envoyer votre message, cochez l'icône de la barre d'outils Signer le message dans la fenêtre d'édition.
Puis, vous pouvez envoyer le message.
L'identité que vous utilisez pour écrire le message courant a besoin d'être connectée à une Clé OpenPGP dans la section Identité de la boîte de dialogue Configuration.
Pour le signer, KMail a besoin de votre phrase-clef PGP.
Si vous n'avez pas sélectionné l'option Conserver la phrase-clef en mémoire dans l'onglet Sécurité de la fenêtre de configuration de KMail, KMail va vous la demander.
Par ailleurs, si vous aviez déjà coché l'option et donné votre phrase-clef, KMail signera le message sans rien vous demander de plus.
Crypter vos messages
Si KMail ne peut trouver de clé correspondante pour un destinataire ou s'il en trouve plusieurs, il affichera une liste de toutes les clés certifiées dans la boîte de dialogue Sélection de clé de cryptage. en utilisant la case à cocher Mémoriser cette décision, vous pouvez enregistrer votre sélection pour les messages futurs.
Comme mentionné ci-dessus, vous ne pourrez pas lire vos propres messages envoyés cryptés si vous ne cochez pas Crypter les messages également avec ma clef publique dans la page des réglages de Sécurité.
Cependant, notez que votre propre clé n'est pas listée dans la boîte de dialogue Confirmation de la clé de cryptage, même si cette option est activée.
Envoyer votre clef publique
Préparez un message à destination de la personne à qui vous voulez envoyer votre phrase-clef.
Cela aura pour effet de joindre votre clef publique au courrier électronique.
Vous pouvez alors envoyer le message.
Souvenez -vous qu'il n'est pas sûr du tout de signer le message pour vous assurer que le destinataire a la clé correcte.
Il peut y avoir une attaque par un individu interposé, quelqu'un qui changerait la clé et signerait le message avec une autre clé.
C'est pourquoi le destinataire devrait comparer la clé attachée en vérifiant le fingerprint de la clé avec celle qu'il a eue de vous d'une manière sécurisée.
Voyez la documentation de PGP pour de plus amples détails.

Réception d'une clé publique
Vous pouvez recevoir une clé publique en pièce jointe, par http, ftp ou une disquette.
Avant de pouvoir l'utiliser pour crypter un message vers l'utilisateur d'une clé, vous devriez vérifier la clé (vérifiez le fingerprint ou regardez les signatures de confiance).
Configurer KMail
La fenêtre de configuration de KMail vous donne plusieurs manières de configurer KMail.
Elle est divisée en six pages, chacune représentée par une des icônes dans la liste sur le côté gauche de la fenêtre.
Ce chapitre va décrire chaque page en détail.
La page Identité
Vous pouvez trouver une brève introduction à la page Identité dans la section Débuter.
Cette page vous permet de créer une ou plusieurs identités, c'est-à-dire des combinaisons de nom, adresse électronique et autres réglages.
Par exemple, vous créerez une identité pour les affaires et une pour le privé.
Si vous avez plusieurs adresses électroniques, vous pouvez créer une identité par adresse.
Vous pourrez sélectionner une adresse dans une base d'adresses.
La page consiste en une liste déroulante nommée Identité, trois boutons pour effectuer des actions générales liées à l'identité et trois onglets supportant les options d'identité.
Pour renommer l'identité sélectionnée actuellement, cliquez sur le bouton Renommer....
Une petite fenêtre de dialogue va apparaître et vous demander le nouveau nom de cette identité.
Veuillez noter que l'identité défaut ne peut être renommée ni supprimée.
La boîte de dialogue Nouvelle identité
Vous pouvez atteindre cette boîte de dialogue par la boîte de dialogue de configuration de KMail.
Vous pouvez écrire le nom de la nouvelle identité dans la zone de saisie de la boîte de dialogue Nouvelle identité.
Ce nom sera donné à la nouvelle identité dans la liste des identités.
Vous pouvez choisir la façon dont la nouvelle identité sera initialisée en validant l'un des trois boutons radio au centre de la boîte de dialogue.
Avec des champs vides
Tous les champs de la nouvelle identité sont vides ou préremplis avec des paramètres par défaut.
Utiliser les paramètres du centre de configuration de KDE.
Dupliquer une identité existante
Copie tous les champs d'une identité existante.
Vous pouvez choisir quelle identité servira de modèle en sélectionnant l'entrée appropriée dans la boîte surgissante Identités existantes.

Nom
Saisissez votre nom complet ici (quelquefois nommé aussi nom d'affichage).
Bien que ce champ ne soit pas obligatoire, il est recommandé de saisir une valeur correcte ici.

Ce champ est purement optionnel.
La plupart des agents utilisateurs afficheront cette information dans un champ séparé Organisation.
Cependant, KMail le met entre parenthèses et l'affiche avec l'adresse électronique dans le champ d'en-tête De.

Options avancées
L'onglet Avancé vous permet de configurer certaines options utilisées rarement ou des paramètres particuliers pour l'identité actuellement sélectionnée.
Adresse de réponse
Écrivez l'adresse à qui les réponses à vos messages électroniques doivent être envoyées.
Ne remplissez ce champ que s'il est différent de votre adresse normale (spécifiée par Nom et Adresse électronique sur l'onglet Général), car les réponses par défaut se font vers l'adresse de l'expéditeur.
Ce champ n'est utile que si vous voulez des réponses à vos courriers vers quelqu'un d'autre que votre adresse régulière, par exemple si vous utilisez cette identité pour envoyer des courriers d'une adresse qui ne peut pas recevoir de messages.
Notez que ce champ sera peut-être abandonné dans les versions ultérieures du standard de l'Internet.
De plus, des listes de diffusion écrasent l'en-tête avec leur propre adresse. pour être sûres que les réponses vont à la liste au lieu des envoyeurs.
Ainsi, l'utilité de ce champ est très limitée et il ne devrait être utilisé que dans de rares cas.
Clef OpenPGP
Sélectionnez ici la clé OpenPGP qui devrait être utilisée pour signer (numériquement) les messages quand cette identité est utilisée.
Quand vous cliquez sur le bouton Changer..., une boîte de dialogue de sélection de clé s'affiche là où vous pouvez sélectionner la clé pour utiliser dans la liste des clés secrètes trouvées.
Si vous n'avez pas de clé OpenPGP ou ne savez pas ce que c'est, vous n'avez pas besoin de vous préoccuper de ce réglage.
La clé sélectionnée sera aussi utilisée comme destinataire quand vous cocherez Toujours crypter les messages envoyés à soi -même sur l'onglet OpenPGP de la page Security.
Vous pouvez aussi avoir besoin de configurer votre environnement OpenPGP d'abord, pour que la boîte de dialogue de sélection fonctionne.
Dossier Envoyés
Sélectionnez le dossier dans lequel vous désirez que les messages soient archivés une fois qu'ils ont été envoyés avec cette identité.
Vous pouvez exercer un contrôle plus fin sur l'endroit où les courriers envoyés doivent être stockés en créant un filtre des messages qui s'applique aux courriers sortants.
Dossier des brouillons
Sélectionnez le dossier dans lequel vous désirez que les brouillons soient archivés en utilisant cette identité.
Transport spécial
Sélectionnez ou saisissez un serveur SMTP de remplacement qui devrait être utilisé lorsque du courrier électronique sera envoyé avec cette identité.
Vous devez configurer les serveurs de courrier sortants d'abord, avant de pouvoir en choisir dans la liste.
Vous pouvez faire ceci sur l'onglet Envoi des messages de la page Réseau.
Signature
Ce type de signature n'a rien à voir avec les signatures numériques pour lesquelles vous pouvez sélectionner la clé à utiliser dans l'onglet Options avancées.
C'est juste une mauvaise appellation d'appeler ceci une signature, mais comme le terme est déjà utilisé partout, nous conservons cette notation.
Gardez simplement à l'esprit que ces signatures et les signatures numériques sont des choses totalement différentes.
KMail peut obtenir des textes de signatures de diverses sources.
La manière traditionnelle sous Unix est de lire le texte dans un fichier nommé .signature dans votre dossier personnel.
Ce fichier peut être partagé entre plusieurs programmes, de telle manière que vous aurez la même signature dans chaque programme de courriers que vous utilisez.
KMail peut aussi lire le texte de la signature dans la sortie d'une commande.
Ainsi, vous pouvez utiliser des programmes comme fortune pour créer un nouveau texte de signature pour chaque courrier.
Tout ce qui imprime sur stdout est capturé et utilisé comme texte de signature.
En troisième possibilité, vous pouvez écrire la signature directement dans la boîte de dialogue de configuration de KMail.
Pour ce faire, sélectionnez Insérer la signature depuis le champ de saisie ci-dessous et écrivez le texte dans la boîte de texte qui apparaît.
Sur l'Internet, les signatures sont par convention séparées du corps par une ligne qui contient seulement trois caractères -- (tiret, tiret, espace).
KMail fera automatiquement précéder le texte de signature de cette ligne si elle n'est pas déjà présente dans le texte de la signature.
La page Réseau
Vous pouvez trouver une introduction rapide à la page Réseau sur la section Configurer votre compte.
Cette page vous permet de créer un ou plusieurs comptes (entrants et sortants), c'est-à-dire des combinaisons de serveurs de courrier, d'informations de connexion et d'autres réglages.
Typiquement, vous créerez un compte sortant (utilisé pour envoyer des courriers) et un entrant (utilisé pour recevoir votre courrier).
Vous pouvez en créer autant que vous voulez, et assigner chacun à une identité différente ou choisir ces options différemment pour chaque courrier.
Envoi
L'onglet Envoi des messages vous permet de définir de nouveaux serveurs de courriers sortants et des options communes.
Confirmer avant l'envoi lancera une boîte de confirmation à chaque fois que vous enverrez un message.
Type d'envoi par défaut vous permet de définir ce qui se produit quand un message est envoyé.

Avec Notification d'arrivée de nouveaux messages, vous pouvez déterminer comment KMail notifiera l'arrivée de nouveaux messages.
Bien sûr, ceci n'a de sens que si Activer la vérification régulière du courrier est activé au moins pour un compte.

Page Apparence
Polices
Cette section vous permet de changer le type, la taille et le jeu de caractères des polices d'affichage.
Texte du message détermine les polices du panneau de lecture, Éditeur détermine les polices pour l'écriture des messages dans la fenêtre de composition.
Il y a une entrée séparée pour Liste des messages - date de telle manière que vous pouvez choisir une police à chasse fixe pour le champ de date afin de faciliter la lisibilité.
Couleurs
Cette section vous permet de changer la couleur du texte.
Réutiliser ces couleurs pour les citations plus profondes signifie que même le texte cité plus de 3 fois apparaît en couleur.
Notez que les couleurs du texte sélectionné ne fonctionne que dans la partie lecture des messages, pas dans l'éditeur.
Disposition
Afficher la liste des dossiers au format long vous permet de basculer la liste du format long au format court.
Le format long montre les dossiers sur toute la hauteur du côté gauche de la fenêtre principale de KMail, ce qui vous permet de voir plus de dossiers, mais prend de la place pour voir le texte des messages.
Afficher la barre colorée active une barre qui donne des informations sur le cryptage et la signature numérique du message à gauche du panneau de lecture.
Avec Afficher la taille des messages, il y aura une autre colonne dans le panneau d'en-têtes qui montre la taille des messages en octets.
Afficher les en-têtes de messages par fil de discussion mettra ensemble tous les messages en une sorte d'arborescence, de telle manière que les réponses à un message sont au-dessous du message.
Avec Options de tri par fil de discussion, vous pouvez sélectionner sous quelles conditions les fils doivent apparaître développés par défaut (c'est-à-dire non repliés).
Vous pouvez toujours développer les fils en utilisant les boutons + / - si vous ne désactivez pas ceci ici.
Avec Affichage de la date, vous pouvez choisir entre plusieurs formats de date.

Général
Préférer le HTML au texte brut vous affichera les messages HTML avec leurs formatage et affichage HTML.
Nous recommandons fortement de laisser cette option désactivée, car des problèmes de sécurité avec le HTML pourraient survenir.

Permettre le téléchargement des références externes depuis l'Internet chargera des images, feuilles de style, etc. externes depuis l'Internet quand vous lisez un message HTML.

Nous recommandons de laisser cette option inactivée.

Le menu Dossier
Dossier Créer...
Ouvre la boîte de dialogue Nouveau dossier qui vous permet de créer un nouveau dossier.
Vous pouvez faire que ce dossier soit un sous-dossier si vous choisissez un dossier parent en utilisant la sélection Appartient à.
Le dossier peut être en format soit mbox, soit maildir. mbox enregistre les messages d'un dossier en un fichier, alors que maildir enregistre chaque message dans son propre fichier. maildir peut être considéré comme plus robuste, mais il peut être plus lent sur certains systèmes.
Notez qu'il n'y a actuellement pas de fonction dans KMail qui vous permet de convertir entre les deux formats automatiquement.

Procmail permet à l'utilisateur de spécifier les fichiers de verrouillage de trois manières différentes, de telle manière qu'il n'y a pas de manière d'établir une correspondance entre les fichiers de spool et les fichiers de verrouillage.

Utiliser d'autres fichiers de boîtes aux lettres avec KMail
KMail offre un outil d'importation pour les messages et carnets d'adresse de certains autres clients de courrier.
Vous pouvez avoir besoin de lire ce chapitre si cet outil ne fonctionne pas pour vous.
Cette section est pour tous les utilisateurs qui ont besoin de déplacer leurs messages d'un précédent client courrier vers KMail.
KMail peut enregistrer ses messages en utilisant le format mbox, un des formats les plus largement utilisés sur les systèmes UNIX.
Les boîtes Mbox stockent les messages dans un fichier, et identifient l'endroit où les messages commencent et finissent par la ligne From (ne mélangez pas ceci avec l'en-tête From: qui contient l'expéditeur du message).
Pour beaucoup de programmes de courrier UNIX, tout ce que vous devez faire est de déplacer vos boîtes aux lettres dans ~/Mail (ou faites un lien symbolique Mail sur le dossier qui contient vos boîtes aux lettres), assurez -vous qu'ils sont autorisés en écriture par votre utilisateur, et lancez KMail.
Les boîtes aux lettes devraient maintenant s'afficher correctement dans KMail.
N'utilisez pas de deuxième client de courrier qui accède aux fichiers dans ~/Mail pendant que KMail fonctionne, ou vous pourriez perdre des messages.
Cette section explique seulement comment importer des boîtes aux lettres vers KMail une fois, ce n'est pas utile si vous comptez utiliser plusieurs clients de courrier pour vos boîtes aux lettres à l'avenir.
Eudora Lite / Eudora Pro
Eudora utilise le format mbox dans ses fichiers de courrier.
Pour les utiliser avec KMail, assurez -vous que vos boîtes aux lettres Eudora ont été purgées, puis copiez les fichiers .mbx (Windows Eudora) ou fichiers de boîte aux lettres Eudora (Mac Eudora) vers votre dossier ~/Mail.
Vous n'avez pas besoin de copier les fichiers index.
Une fois que vous lancez KMail, les boîtes aux lettres devraient apparaître dans le panneau des dossiers et les messages dans le panneau des en-têtes.
Si les messages n'apparaissent pas dans le panneau des en-têtes, votre boîte aux lettres peut toujours contenir des caractères de fin de ligne de Windows ou Mac.
Utilisez votre éditeur de texte favori, la commande recode ou un langage de script pour changer les caractères de fin de ligne de Windows ou Mac en fin de lignes UNIX.
MMDF
Ce format est assez proche du format mailbox pour que KMail puisse utiliser ces boîtes aux lettres si vous les copiez simplement dans votre dossier ~/Mail.
Cependant, les boîtes aux lettres MMDF n'ont pas été testées avec KMail, ainsi, vos résultats peuvent varier.
Si vous pouvez obtenir ce format pour travailler dessus avec KMail, veuillez nous le faire savoir de telle manière que nous puissions inclure plus d'informations spécifiques dans la prochaine version de la documentation.
Boîtes aux lettres MH
Les boîtes aux lettres MH sont des dossiers qui contiennent des fichiers correspondant à chaque message dans cette boîte aux lettres.
Un script shell pour convertir les boîtes aux lettres MH en mbox, mh2kmail,, est inclus au moins dans le source de KMail, mais peut-être pas dans la version exécutable.
Lancer ce script sur un dossier MH le convertira en fichier mbox.
Nous recommandons fortement de sauvegarder vos dossiers MH avant d'utiliser ce script.

Pegasus Mail
Pegasus pour win32 utilise de simples fichiers pour les dossiers de courrier, semblables à KMail.
Les fichiers de dossiers Pegasus ont l'extension .pmm, mais ce sont de simples fichiers mbox, à l'exception que les messages ne commencent pas par l'en-tête From, mais par un caractère de contrôle.
Pour corriger cela, remplacez chaque instance du caractère de contrôle par From aaa@aaa Mon Jan 01 00:00:00 1997.
Cette ligne From devrait être la première ligne de chaque message avant Received: et les autres en-têtes.
Assurez -vous d'utiliser un éditeur de texte qui vous permet d'enregistrer les fichiers dans le format UNIX ou créez de nouveaux dossiers dans Pegasus qui sont en format UNIX et copiez -y vos courriers.
