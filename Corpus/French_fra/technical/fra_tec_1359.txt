Données (pour les documents formulaire XML)
L'onglet Données de la boîte de dialogue Propriétés d'un document de formulaire XML permet des paramétrages de formulaires XML.
Données
d'un contrôle dépendent du contrôle correspondant.
Vous ne verrez que les options qui sont disponibles pour le contrôle et le contexte actifs.
Les champs suivants sont disponibles :
Modèle de données XML
Sélectionnez un modèle dans la liste de tous les modèles dans le document actif.
Liaison
Sélectionnez ou saisissez le nom d'une liaison.
La sélection du nom d'une liaison existante associe la liaison avec le contrôle de formulaire.
La saisie d'un nouveau nom crée une nouvelle liaison et l'associe au contrôle de formulaire.
Expression de liaison
Saisissez le noeud DOM avec lequel lier le modèle de contrôle.
Cliquez sur le bouton ... pour ouvrir une boîte de dialogue dans laquelle vous saisissez l'expression XPath.
Obligatoire
Spécifie si l'élément doit être inclus dans le XForm.
Important
Déclare l'élément comme important.
Lecture seule
Déclare l'élément en lecture seule.
Contrainte
Déclare l'élément comme une contrainte.
Calcul
Déclare que l'élément est calculé.
Type de données
Sélectionnez un type de données contre lequel le contrôle doit être validé.
x
Sélectionnez un type de données défini par l'utilisateur et cliquez sur le bouton pour supprimer le type de données défini par l'utilisateur.
+
Cliquez sur le bouton pour ouvrir une boîte de dialogue dans laquelle vous pouvez saisir le nom d'un nouveau type de données défini par l'utilisateur.
Le nouveau type de données hérite de toutes les facettes du type de données actuellement sélectionné.
La liste suivante liste toutes les facettes qui sont valides pour les types de données.
Certaines facettes ne sont disponibles que pour certains types de données.
Espaces
Spécifie comment les espaces vides doivent être gérés lorsqu'une chaîne du type de données actif est exécutée.
Les valeurs possibles sont Préserver, Remplacer et Réduire.
La sémantique suit la définition décrite à http://www.w3.org/TR/xmlschema-2/#rf-whiteSpace.
Motif
Spécifie un motif d'expression régulière.
Les chaînes validées contre le type de données doivent être conformes à ce motif pour être valides.
La syntaxe du type de données XSD des expressions régulières est différente de la syntaxe des expressions régulières utilisées ailleurs dans %PRODUCTNAME, par exemple dans la boîte de dialogue Rechercher et remplacer.
Chiffres (total)
Spécifie le nombre total maximum de chiffres que les valeurs de type décimales peuvent avoir.
Chiffres (fraction)
Spécifie le nombre total maximum de fraction de chiffres que les valeurs de type décimales peuvent avoir.
Max.
(inclusif)
Spécifie un lien supérieur inclusif pour les valeurs.
Max.
(exclusif)
Spécifie un lien supérieur exclusif pour les données.
Min.
(inclusif)
Spécifie un lien inférieur inclusif pour les valeurs.
Min.
(exclusif)
Spécifie un lien inférieur exclusif pour les valeurs.
Longueur
Spécifie le nombre de caractères d'une chaîne.

Longueur (minimale)
Spécifie le nombre minimum de caractères d'une chaîne.
Longueur (maximale)
Spécifie le nombre maximum de caractères d'une chaîne.
