Установка
Я, бывает, теряю терпение после сбоя fsck (Линус Торвальдс после обнаружения, что одна из его файловых систем уничтожена новой бета-версией ядра.)
Не волнуйтесь! Использование бета-версий программного обеспечения может иногда вызвать значительное расстройство нервной системы. Однако вряд ли установка KDE выведет из строя ваш жесткий диск. На самом деле, эта глава написана, чтобы помочь вам в процессе инсталляции и дать вам возможность увидеть ваш новый рабочий стол в среде KDE настолько быстро и с наименьшими бедами на вашу голову, насколько это возможно. Также как и при установке любого менеджера окон, рекомендуется, чтобы вы сделали резервную копию всех ваших файлов настройки, касающихся X11, до того как вы начнете установку. Если вы не знаете где эти файлы находятся - - просмотрите все скрытые файлы (.*) в вашем домашнем каталоге и каталоге / usr / X11 / lib / X11 / xdm.
Требования
Перед установкой KDE убедитесь, что ваша система отвечает следующим требованиям:
Ваша система UNIX совместима со стандартом POSIX.
UNIX 'ы, про которые известно, что KDE в них работает, включают:
Linux, FreeBSD, Solaris, HP-UX и MkLinux. Мы работаем над тем, чтобы отладить KDE и для других систем.
Свободное меcто на жестком диске там, где будет создан каталог / opt / kde. Мы рекомендуем зарезервировать примерно 80 МБ, если ваша система поддерживает разделяемые библиотеки, или значительно больше, если не поддерживает. Если вам нужно (или хочется) скомпилировать KDE из исходников, не забудьте зарезервировать 100 или более мегабайт в / usr / src.
Работающая система X11 с или без xdm. Если вы все еще не установили систему X Window, первым делом проверьте ваш дистрибутив UNIX 'а на наличие установочной версии этой системы. Если вы не можете найти таковую - - зайдите на сайт XFree86, чтобы узнать, как получить и установить систему X Window, или свяжитесь со службой поддержки пользователей вашего дистрибутива UNIX.
Библиотеку Qt, версии 3.01 или выше. Вы можете найти ее на FTP-сервере компании Troll Tech как в rpm, так и tgz форматах.

Использование TGZ для установки на другие системы
Если ваш дистрибутив Linux не поддерживает форматы RPM или DEB для установочных архивов, или вы вообще не используете Linux, вы должны компилировать KDE сами. В будущем мы планируем поставлять бинарный дистрибутив, который будет иметь свою собственную программу инсталляции.
Для того, чтобы правильно собрать и установить KDE, вам нужно установить на ваш жесткий диск следующие программы:
Компилятор ANSI-C, например, GNU C компилятор.
Компилятор ANSI-C+ + например, GNU C++.
Программу make.
Qt версии 3.01 или выше.
X11 для разработчиков (include-файлы часто отсутствуют в дистрибутиве)
kdesupport - version .tar.gz
kdelibs - version .tar.gz
kdebase - version .tar.gz
и все прочие пакеты, которые вы хотите установить. Мы советуем, по меньшей мере, взять kdeutils.
Везде последняя версия означает номер новейшей версии. Когда вы скачате все необходимое, разархивируйте все эти файлы в / usr / src. В результате вы должны получить следующую структуру каталогов:
/usr / src / kdesupport
/usr / src / kdelibs
/usr / src / kdebase
/usr / src /... (любые другие пакеты)
Убедитесь, что у вас есть разрешение на запись в каталог / opt / kde.
Далее вы должны собрать и установить пакеты:
cd в каталог пакета, который вы хотите установить (см. выше)
. / configure (c нужными вам параметрами)
make
make install
Повторите все эти шаги для каждого пакета, который вы хотите установить.
А вообще-то эти инструкции могут быть использованы почти для каждого пакета с исходниками, не только пакета KDE.
Некоторые параметры не критичны и используются лишь для обхода некоторых известных проблем (например, --disable-FEATURE). Но некоторые параметры действительно важны, такие как --with-shadow, поэтому лучше ознакомиться с имеющимися параметрами до компиляции.
Если у вас возникли проблемы, которые вы не смогли решить, отправьте копию файла config.log по адресу, указанному в файле README из неподдающегося пакета.
Вы должны устанавливать KDE в следующем порядке: kdesupport, kdelibs, затем пакеты приложений (например, kdebase). Все эти приложения должны зависеть только от kdelibs, так что вы можете их компилировать все одновременно (если у вас мощная машина).

Для Linux большинство необходимых средств для сборки KDE можно найти на ftp: / /sunsite.unc.edu / pub / Linux / GCC или на ftp: / /sunsite.unc.edu / pub / GNU. Библиотеки Qt можно скачать с ftp: / /ftp.troll.no / pub / qt / linux в виде исходников или пакетов RPM для Linux. В обоих вариантах имеется детальная информация по установке. Требуемые изменения в файлах настройки
Поместив бинарники KDE в нужный каталог, поправьте еще свои стартовые скрипты.
Следующий рецепт был проверен на SuSE Linux 5.0, и он должен работать на других дистрибутивах UNIX. Настоятельно рекомендуется делать копии файлов настройки, прежде чем менять их! Добавьте в / etc / profile следующее:
export PATH=$PATH: / opt / kde / bin export KDEDIR= / opt / kde
Отредактируйте файл .xinitrc в своем домашнем каталоге. Найдите команду вызова оконного менеджера и замените ее на startkde.
Теперь же, когда все готово для запуска KDE, прочитайте главу 4, где описывается ход первого запуска KDE. Если что-то не получилось, имеет смысл попробовать собрать KDE самому. Прочитайте раздел Использование TGZ для установки на другие системы, где это описано.
