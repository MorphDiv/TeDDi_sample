Использование интернет-трансляции ASP
Интернет-трансляция может использоваться сразу после того, как экспортированные файлы становятся доступны с сервера HTTP.
Пример
:
Предположим, что Microsoft Internet Information Server установлен на данном компьютере.
Директория для HTML файлов &quot; c:\Inet\wwwroot\presentation &quot; .
URL компьютера: &quot; http://myserver.com &quot; .
Файлы, созданные в процессе экспорта, сохраняются в каталоге c:\Inet\wwwroot\presentation\.
В этом каталоге функция экспорта создает файл HTML с именем, например, &quot; secret.htm &quot; .
Это имя указывается в диалоговом окне &quot; Сохранить &quot; (см. выше).
Теперь докладчик может просматривать файлы экспорта HTML, вводя URL-адрес http://myserver.com/presentation/secret.htm в любой браузер HTTP с поддержкой JavaScript.
Кроме того, появляется возможность изменить страницы с помощью некоторых элементов управления формами.
Просмотр слайдов, выбранных докладчиком, может осуществляться с помощью URL-адреса http://myserver.com/presentation/webcast.asp.
Чтобы перейти к другим слайдам на этом URL-адресе аудитории, необходимо знать имена файлов.
Убедитесь, что на сервере HTTP не отображается список каталогов.

Использование интернет-трансляции Perl
Файлы, созданные при экспорте, должны быть установлены на сервере HTTP с поддержкой Perl.
Это нельзя сделать автоматически из-за существования множества различных серверов HTTP, поддерживающих Perl.
Действия, которые следует выполнить, описаны ниже.
За сведениями о применении этих действий к конкретному серверу обратитесь к документации сервера или к администратору сети.
Сначала необходимо переместить файлы, созданные при экспорте, в нужный каталог на сервере HTTP.

URL для сценариев Perl
.
Этот каталог настраивается таким образом, чтобы находящиеся в нем сценарии Perl могли запускаться по запросу HTTP.
В системах UNIX файлы с расширением pl становятся исполняемыми на сервере HTTP.
Как правило, это выполняется с помощью команды chmod.
Файл currpic.txt может получить право записи на сервер HTTP.
После выполнения этих действий можно использовать интернет-трансляцию.

Теперь скопируйте все файлы *.htm, *.jpg и *.gif из каталогов, указанных при экспорте, в каталог //user/local/http/presentation/ на HTTP-сервере, а все файлы с расширениями *.pl и *.txt скопируйте в каталог //user/local/http/cgi-bin/.
Войдите на сервер как в корневой каталог и перейдите к каталогу //user/local/http/cgi-bin/.
Соответствующие права можно определить с помощью команды chmod.
