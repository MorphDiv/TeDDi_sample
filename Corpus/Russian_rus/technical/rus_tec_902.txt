Специальные настройки
Задает способ работы с данными в базе.
Правка – База данных – Дополнительные параметры
.
Доступность следующих элементов управления зависит от типа базы данных.
Использовать соглашение об именах SQL92
Разрешает использовать в именах источников данных только те символы, которые соответствуют соглашению по присвоению имен SQL92.
Все прочие символы отклоняются.
Каждое имя должно начинаться с прописной или строчной буквы либо со знака подчеркивания (_).
Остальные символы могут быть буквами ASCII, цифрами или знаками подчеркивания.
Завершение строк текста с помощью CR + LF
Для завершения каждой строки текста выберите кодовую пару CR + LF (предпочтительно для операционных систем DOS и Windows).
Добавить псевдоним таблицы для инструкции SELECT
Добавляет псевдоним к имени таблицы в инструкции SELECT.
Использовать синтаксис внешнего соединения &quot; {OJ } &quot;
Для внешних соединений используйте управляющие последовательности.
Такая управляющая последовательность использует синтаксис {oj внешнее_соединение}
Пример:
select Article.* from {oj item LEFT OUTER JOIN orders ON item.no=orders.ANR}
Игнорировать разрешения драйвера базы данных
Игнорирует права доступа, предоставляемые драйвером базы данных.
Заменить именованные параметры на &quot; ? &quot;
Заменяет именованные параметры в источнике данных на вопросительный знак (?).
Отобразить столбцы версии (если доступны)
Отображает внутренний номер версии записи в таблице базы данных.
Использовать имя каталога в инструкции SELECT
Использование текущего источника данных каталога.
Этот параметр удобно использовать, если источником данных ODBC является сервер базы данных.
Не выбирайте этот параметр, если источником данных ODBC является драйвер dBASE.
Использовать имя схемы в инструкции SELECT
Позволяет использовать имя схемы в инструкции SELECT.
Создать указатель с инструкциями ASC и DESC
Создает указатель с инструкцией ASC или DESC.
Сравнение логических значений
Выберите тип сравнения логических значений, который требуется использовать.
Проверка ввода данных в требуемые поля формы
Если при вводе в форму новой записи или обновлении существующей оставляется пустым поле, связанное с столбцом базы данных, требующим ввода, выводится сообщение с требованием заполнить пустое поле.
Если этот флажок не установлен, формы в текущей базе данных не проверяются на требуемые поля.
Флажок доступен для всех типов источников данных, поддерживающих доступ к данным для записи.
Флажок отсутствует для электронных таблиц, текста, csv и различных, открытых в режиме только для чтения, адресных книг .
Игнорировать информацию поля валюты
Только для JDBC-соединений Oracle.
Если включено, то ни один столбец не рассматривается как поле валюты.
Тип поля, возвращаемый из драйвера базы данных, не учитывается.
Использование литералов даты/времени, совместимых с ODBC
Установите этот флажок для использования литералов даты/времени, совместимых с открытым интерфейсом взаимодействия с базами данных.
