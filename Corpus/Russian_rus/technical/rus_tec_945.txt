Вид - Навигатор отчетов
.
В навигаторе отчетов представлена структура отчета.
Навигатор отчетов можно использовать для вставки функций в отчет.
Щелкните по записи в навигаторе отчетов.
В ракурсе &quot; Построитель отчетов &quot; выбирается соответствующий объект или область.
Для открытия контекстного меню щелкните правой кнопкой мыши по записи.
Ввод функций в отчет
В контекстном меню навигатора отчетов отображаются те же команды, что и в ракурсе &quot; Построитель отчетов &quot; , плюс дополнительные команды для создания или удаления новых функций.

Вычисление суммы для каждого клиента
Откройте навигатор отчетов.
Откройте запись &quot; Группы &quot; и группу, где требуется вычислить затраты.
В группе имеется подчиненная запись с названием &quot; функции &quot; .
Откройте (щелчком правой кнопкой) контекстное меню записи функций, выберите создание новой функции и выделите ее.
Функция отображается в браузере свойств.
Измените имя, например, на CostCalc, а формулу - на [CostCalc] + [введите имя столбца затрат].
В качестве начального значения введите 0.
Теперь можно вставить текстовое поле и связать его с [CostCalc] (выводится в списке поля данных).
Для значения поля может потребоваться задание начального значения, например [поле].
Если в столбце затрат имеются пустые поля, для замены содержимого пустых полей на нули используется следующая формула:
[SumCost] + IF(ISBLANK([поле]);0;[поле])

Formula
Ввод формулы, определяющей функцию.
Используется синтаксис OpenFormula.
Initial value
Ввод начального значения для анализа формулы.
Как правило, это значение 0 или 1.
Deep traversing
При активном режиме глубокого прослеживания функции анализируются с учетом всех нижних уровней иерархии.
Например, этот режим может использоваться для нумерации строк.
Если режим глубокого прослеживания не активен, анализируется только первый уровень иерархии.
Pre evaluation
Если активирован режим предварительного анализа, расчет по функциям выполняется только после завершения отчета.
