Данные (для документов XML Form)
На вкладке &quot; Данные &quot; диалогового окна &quot; Свойства &quot; для документа XML Form предлагаются некоторые настройки XML-форм.
Данные
элемента управления зависят от конкретного элемента управления.
Отображаются только те параметры, которые могут применяться в отношении текущего элемента и в текущем контексте.
Доступны следующие поля:
Модель данных XML
Выберите модель из списка моделей в текущем документе.
Привязка
Выберите или введите имя привязки.
Выбор имени существующей привязки устанавливает соответствие между привязкой и элементом управления.
Ввод нового имени создает новую привязку и устанавливает ее соответствие с элементом управления.
Выражение связывания
Введите узел DOM, который должен быть связан с моделью элемента управления.
Нажмите кнопку ... для ввода выражения XPath в диалоговое окно.
Требуется
Указывает, следует ли включить данный элемент в XForm.
Относящийся
Определяет элемент как релевантный.
Только для чтения
Определяет элемент как предназначенный только для чтения.
Ограничение
Определяет элемент как ограничение.
Вычисление
Определяет элемент как вычисляемый.
Тип данных
Выберите тип данных, по которому будет осуществляться проверка элемента управления.
x
Выберите пользовательский тип данных и нажмите эту кнопку для его удаления.
+
Эта кнопка используется для открытия диалогового окна, в котором можно ввести имя нового пользовательского типа данных.
Новый тип данных наследует все настройки типа данных, выбранного в данный момент.
Далее перечисляются все настройки, действительные для типов данных.
Некоторые настройки доступны только для определенных типов данных.
Пробельные символы
Используется для определения способа обработки пробелов при обработке строки данных текущего типа.
Возможные значения: &quot; Сохранить &quot; , &quot; Заменить &quot; и &quot; Удалить &quot; .
Семантика соответствует определению, приведенному по адресу http://www.w3.org/TR/xmlschema-2/#rf-whiteSpace.
Шаблон
Определяет шаблон регулярного выражения.
Строки, проверяемые на допустимость по типу данных, должны соответствовать этому шаблону.
Синтаксис типа данных XSD для регулярных выражений отличается от синтаксиса регулярного выражения, используемого в других компонентах %PRODUCTNAME, например, в диалоговом окне &quot; Найти и заменить &quot; .
Разрядов (всего)
Определяет максимальное общее число разрядов, которые могут иметь значения данных десятичного типа.
Разрядов (дробная часть)
Определяет максимальное общее число дробных разрядов, которые могут иметь значения данных десятичного типа.
Максимум (включая)
Определяет включаемый в диапазон верхний предел значений.
Максимум (не включая)
Определяет исключаемый из диапазона верхний предел значений.
Минимум (включая)
Определяет включаемый в диапазон нижний предел значений.
Минимум (не включая)
Определяет исключаемый из диапазона нижний предел значений.
Длина
Указывает количество символов для строк.

Указывает минимальное количество символов для строк.
