Общие
Получить созданные значения
Включение в $[officename] поддержки автоинкрементных полей данных для текущего источника данных ODBC или JDBC.
Установите этот флажок, если база данных не поддерживает автоинкрементный компонент в слое SDBCX.
Вообще автоинкремент выбирается для поля первичного ключа.
Инструкция AUTO_INCREMENT
Введите указатель команд SQL, задающий для источника данных автоинкремент указанного целочисленного поля данных.
Например, типичная инструкция SQL для создания поля данных выглядит следующим образом:
CREATE TABLE &quot; table1 &quot; ( &quot; id &quot; INTEGER)
Чтобы задать автоинкремент для поля &quot; id &quot; в базе данных MySQL, измените эту инструкцию следующим образом:
CREATE TABLE &quot; table1 &quot; ( &quot; id &quot; INTEGER AUTO_INCREMENT)

Запрос созданных значений
Введите инструкцию SQL, возвращающую последнее автоинкрементное значение для поля данных первичного ключа.
Например:
SELECT LAST_INSERT_D();
Использовать соглашение об именах SQL92
Допустимы только имена, в которых используются символы, удовлетворяющие соглашению об именах SQL92 в источнике данных.
Все другие символы отбрасываются.
Каждое имя должно начинаться со строчной или прописной буквы или с подчеркивания ( _ ).
Остальные символы должны представлять собой буквы, подчеркивания и цифры ASCII.
Использовать каталог для файловых баз данных
Использование текущего источника данных каталога.
Это удобно, если источник данных ODBC - сервер базы данных.
Если источником данных ODBC является драйвер dBASE, устанавливать этот флажок не следует.
