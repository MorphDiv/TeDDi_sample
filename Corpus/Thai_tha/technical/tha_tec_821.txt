การติดตั้ง GNOME
คุณสามารถทดลองใช้ GNOME 2.22 ได้จาก LiveCD ซึ่งมีซอฟต์แวร์ทั้งหมดที่อยู่ใน GNOME 2.22 รวมอยู่ในซีดีแผ่นเดียว คุณสามารถใช้ LiveCD บูตเครื่องโดยตรง โดยไม่ต้องติดตั้งอะไรเลยก็ได้ LiveCD สามารถดาวน์โหลดได้จาก
ในการติดตั้งหรือปรับรุ่นเป็น GNOME 2.22 เราขอแนะนำให้คุณติดตั้งแพ็กเกจอย่างเป็นทางการจากผู้ผลิตหรือผู้แจกจ่ายของคุณ ดิสทริบิวชันหลัก ๆ จะเตรียม GNOME 2.22 ให้คุณเร็ว ๆ นี้ และบางดิสทริบิวชันก็อาจมี GNOME 2.22 รุ่นพัฒนาอยู่แล้วก็ได้ คุณสามารถดูรายชื่อของดิสทริบิวชันที่มี GNOME พร้อมทั้งรุ่นล่าสุดที่มีได้จากหน้า
ถ้าคุณกล้าและอดทนพอ และอยากจะคอมไพล์ GNOME จากซอร์สโค้ดเอง เราขอแนะนำให้ใช้เครื่องมือคอมไพล์อย่างใดอย่างหนึ่ง คือ
ถึงแม้การคอมไพล์ GNOME โดยตรงจาก tarball จะทำได้เหมือนกัน แต่เราขอแนะนำให้ใช้ชุดเครื่องมือคอมไพล์ข้างต้นจะดีกว่า

มีอะไรใหม่สำหรับนักพัฒนา
แพลตฟอร์ม GNOME 2.22 สำหรับนักพัฒนา ให้ฐานที่มั่นคงสำหรับนักพัฒนาซอฟต์แวร์ทั่วไป ในการสร้างโปรแกรมประยุกต์ต่าง ๆ GNOME และแพลตฟอร์ม ใช้สัญญาอนุญาตที่อนุญาตให้ใช้สร้างทั้งซอฟต์แวร์เสรีและเชิงพาณิชย์ เพื่อทำงานบน GNOME ได
ไลบรารีต่างๆ ในแพลตฟอร์ม GNOME จะรับประกันความคงที่ของ API และ ABI ไปตลอดการปล่อยรุ่นต่าง ๆ ของ GNOME 2.x แต่ไลบรารีต่าง ๆ ในตัวเดสก์ท็อป GNOME เอง ไม่มีการรับประกันนี้ แต่โดยส่วนใหญ่ก็จะสอดคล้องกันจากรุ่นถึงรุ่น
GVFS และ GIO
GVFS เป็นระบบแฟ้มเสมือนที่ทำงานในระดับผู้ใช้ โดยมีแบ็กเอนด์รองรับโพรโทคอล SFTP, FTP, DAV, SMB, ObexFTP โดย GVFS เป็นโปรแกรมที่มาแทน GNOME-VFS ซึ่งถือว่าล้าสมัย และไม่ควรถูกเรียกใช้ในโปรแกรมใหม่ ๆ อีกต่อไป
GVFS ประกอบด้วย 2 ส่วน:
GIO เป็นไลบรารีตัวใหม่ซึ่งเป็นส่วนหนึ่งของ GLib ทำหน้าที่เป็น API ของ GVFS
ตัว GVFS เอง เป็นแพกเกจใหม่ที่รวมเอาแบ็กเอนด์สำหรับระบบแฟ้มและโพรโทคอลต่าง ๆ เช่น SFTP, FTP, DAV, SMB และ ObexFTP
GVFS/GIO มีเป้าหมายในการสร้างระบบแฟ้มเสมือน (VFS) ที่ใช้งานได้ง่ายและทันสมัย โดยการสร้าง API ที่นักพัฒนาสามารถเรียกใช้ได้แทน POSIX IO call ได้ แต่แทนที่จะลอกแบบ API ของ POSIX IO มา GIO กลับสร้างส่วนเชื่อมต่อในระดับสูง โดยมีตัวเอกสารเป็นศูนย์กลางขึ้นมา นอกจากการอ่านและเขียนแฟ้มแล้ว GIO ยังมีสาธารณูปโภคในการติดตามแฟ้ม, IO แบบ asynchronous และการเติมเต็มชื่อแฟ้มอีกด้วย
GVFS ทำงานโดยเรียกใช้ดีมอนหลักเพียงตัวเดียว (gvfsd) ซึ่งจะคอยตรวจสอบการเมานท์ GVFS ที่มีอยู่ในขณะนั้น การเมานท์แต่ละอันจะแยกเป็นดีมอนคนละตัว (การเมานท์บางอย่างอาจใช้ดีมอนร่วมกันได้ แต่ส่วนมากจะไม่) ฝั่งลูกข่ายจะคุยกับจุดที่เมานท์โดยใช้ทั้งการเรียกผ่าน D-Bus (ทั้งบัสของ session และ peer-to-peer D-Bus) และโพรโทคอลพิเศษสำหรับข้อมูลในแฟ้ม การแยกส่วนแบ็กเอนด์ออกจากโพรเซสหลัก ช่วยลดความขึ้นต่อกันระหว่างโปรแกรมต่าง ๆ ลง และส่งผลให้ระบบมั่นคงขึ้นในภาพรวม
GVFS ยังมีจุดเมานท์แบบ FUSE ใน
สิ่งที่ต่างจาก GNOME-VFS คือการเชื่อมต่อใน GVFS จะจำสถานะด้วย ซึ่งแปลว่าผู้ใช้ป้อนรหัสผ่านเพียงครั้งเดียวก็พอ ไม่จำเป็นต้องป้อนรหัสผ่านทุกครั้งเมื่อมีการเชื่อมต่ออย่างที่เคยเป็น
เมื่อเปลี่ยนมาใช้ GVFS แล้ว การเมานท์อัตโนมัติและการเรียกโปรแกรมอัตโนมัติในสื่อจะถูกจัดการโดยตรงผ่าน

PolicyKit เป็นกรอบการทำงานด้านการรักษาความปลอดภัยข้ามเดสก์ท็อปตัวใหม่ เป้าหมายของ PolicyKit คือสร้างวิธีมาตรฐานสำหรับโปรแกรมต่าง ๆ ในการขอสิทธิ์ดูแลระบบเพิ่มเติม
ผู้ดูแลระบบสามารถตั้งค่าที่ใช้ได้ทั่วทั้งระบบด้วย PolicyKit ตัวอย่างเช่น คุณอาจอนุญาตให้ผู้ใช้บางคนปรับวันที่และเวลาของนาฬิกา หรือคุณอาจมอบสิทธิ์ให้ผู้ใช้เปิดหรือปิดบริการบางอย่างของระบบ เป็นต้น

เกริ่นนำ
GNOME 2.22 คือรุ่นล่าสุดของ GNOME ระบบเดสก์ท็อปยอดนิยมและหลากแพลตฟอร์ม จุดมุ่งหมายของ GNOME อยู่ที่ความสะดวกใช้ ความเสถียร และการรองรับภาษาทั่วโลกอย่างดีที่สุด รวมทั้งสิ่งอำนวยความสะดวกสำหรับผู้บกพร่องทางกายภาพ GNOME เป็นซอฟต์แวร์เสรีและโอเพนซอร์ส และได้ตระเตรียมเครื่องมือทั่วไปที่ผู้ใช้คอมพิวเตอร์ต้องการจากระบบคอมพิวเตอร์สมัยใหม่ ไม่ว่าจะเป็นอีเมล กรุ๊ปแวร์ เว็บเบราว์เซอร์ การจัดการแฟ้ม มัลติมีเดีย และเกมต่าง ๆ นอกจากนี้ GNOME ยังเป็นแพลตฟอร์มที่ยืดหยุ่นและทรงพลังสำหรับนักพัฒนาซอฟต์แวร์ ทั้งในระดับเดสก์ท็อปและอุปกรณ์พกพา
เดสก์ท็อป GNOME มีการออกรุ่นใหม่ทุก ๆ หกเดือน และนำความสามารถและการปรับปรุงใหม่ ๆ การแก้ข้อผิดพลาด และการแปลข้อความ สู่ผู้ใช้ในทุก ๆ รุ่น และ GNOME 2.22 ก็ยังคงออกตามกำหนดนี้ หากต้องการทราบข้อมูลเพิ่มเติมเกี่ยวกับ GNOME และจุดเด่นที่ทำให้ GNOME แตกต่างจากระบบเดสก์ท็อปคอมพิวเตอร์อื่น ๆ (เช่น ความสะดวกใช้ สิ่งอำนวยความสะดวกสำหรับผู้บกพร่องทางกายภาพ การรองรับนานาภาษา และการให้เสรีภาพ) กรุณาอ่านที่หน้า

GNOME 2.22 รวมเอาการปรับปรุงทุกอย่างใน GNOME 2.20 และก่อนหน้านี้ไว้ทั้งหมด คุณสามารถอ่านความเปลี่ยนแปลงที่เกิดขึ้นใน GNOME 2.20 ได้จาก
