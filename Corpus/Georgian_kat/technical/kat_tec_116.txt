HTTP პროქსის გამოყენება
პროქსის პარამეტრების ჩართვა ინტერნეტით HTTP წვდომისას.
HTTP პროქსის ჰოსტის სახელი
პროქსი სერვერის მანქანური სახელი HTTP-სთვის.
HTTP პროქსის პორტი
"/system/http_proxy/host" კოდით განსაზღვრული მანქანური პორტი პროქსისთვის.
პროქსი სერვერის კავშირების აუთენტიფიკაცია
თუ ჭეშმარიტია, პროქსი სერვერთან კავშირი აუთენტიფიკაციას საჭიროებს. username/password combo მითითებულია "/system/http_proxy/authentication_user" და "/system/http_proxy/authentication_password" საშუალებით.
HTTP პროქსის მომხმარებლის სახელი
მომხმარებლის სახელი HTTP პროქსი სერვერისთვის.
HTTP პროქსის პაროლი
აუთენტიფიკაციის პაროლი HTTP პროქსი სერვერისთვის.
არა პროქსი ჰოსტები
ეს კოდი შეიცავს ჰოსტების სიას, რომელიც პირდაპირაა დაკავშირებული და არა პროქსის საშუალებით (თუ აქტიურია). მნიშვნელობები შესაძლოა იყოს ჰოსტის სახელები, დომენები (სპეცსიმბოლოების გამოყენებით, მაგ., *.foo.com), ჰოსტის IP მისამართები (ორივე, IPv4 და IPv6) და ქსელური მისამართები ქსელის თარგით (მაგალითად 192.168.0.0/24).
პროქსის კონფიგურაციის რეჟიმი
პროქსის კონფიგურაციის რეჟიმის მითითება. შესაძლო მნიშვნელობებია "none" ("არა"), "manual" ("ხელით"), "auto" ("ავტომატური").
უსაფრთხო HTTP პროქსის ჰოსტის სახელი
პროქსი სერვერის მანქანური სახელი უსაფრთხო HTTP-სთვის.
უსაფრთხო HTTP პროქსის პორტი
"/system/proxy/secure_host" კოდით განსაზღვრული მანქანური პორტი პროქსისთვის.
FTP პროქსის ჰოსტის სახელი
პროქსი სერვერის მანქანური სახელი FTP-სთვის.
FTP პროქსის პორტი
"/system/proxy/ftp_host" კოდით განსაზღვრული მანქანური პორტი პროქსისთვის.
SOCKS პროქსის ჰოსტის სახელი
მანქანური სახელი პროქსი არხისთვის.
SOCKS პროქსის პორტი
"/system/proxy/socks_host" კოდით განსაზღვრული მანქანური პორტი პროქსისთვის.

URL რომელშიც მოცემულია პროქსის კონფიგურაციის მნიშვნელობები.
SMB სამუშაო ჯგუფი
Windows ქსელის სამუშაო ჯგუფი ან დომენი რომელშიც მომხმარებელია გაერთიანებული. სამუშაო ჯგუფის შეცვლის ძალაში შესასვლელად აუცილებელია სისტემიდან გამოსვლა და ხელახალი შესვლა.
