兼容性
指定文本文档的兼容性设置。这些选项有助于在导入 Microsoft Word 文档时微调 %PRODUCTNAME。
这里定义的某些设置仅对当前文档有效，必须为每一个文档单独定义。

文档格式化应用打印机标准
指定在打印以及格式化屏幕显示时应用的打印机标准。
如果未选中该框，则格式化屏幕显示和打印时将使用与打印机无关的版式。
如果在当前文档中设置了该选项并保存了此文档，例如以早期的二进制格式保存，此选项将不会被保存。日后以早期格式打开此文件，此选项将被设为默认值。
在段落和表格之间添加间距（当前文档中）
在 $[officename] Writer 中，段落间距的定义与 MS Word 文档有所不同。如果定义了两个段落或表格间的间距，那么在相应的 MS Word 文档中也会添加此间距。
指定在 $[officename] Writer 的文本文档的段落和表格之间是否添加与 MS Word 兼容的间距。
在页面的顶部添加段落和表格的间距（当前文档中）
指定如果一个段落在文档的首页，在页面顶部的段落间距是否在页面或列的起始位置也有效。
同样的效果应用于分页。
如果您导入一个 MS Word 文档，在转换文档时就会自动扩大段落之间的间隔。
使用 StarSuite 6.0/7 制表位格式
指定如何在右页边距内将文本对齐制表位、指定如何处理十进制制表位、指定如何处理靠近换行处的制表位。
如果未选中此复选框，制表位的处理方式与其他 Office 应用程序一样。
在由您当前版本的 Writer 创建的文本文档中，默认使用新的制表位处理。在由 StarSuite 8 或 OpenOffice.org 2.0 之前版本的 Writer 创建的文本文档中，使用旧的制表位处理。
在多行文字之间不添加前导（多余）空格
指定不在文本行间添加附加前导（额外间距），即使所使用的字体包含附加前导属性。
在由您当前版本的 Writer 创建的文本文档中，默认使用附加前导。在由 StarSuite 8 或 OpenOffice.org 2.0 之前版本的 Writer 创建的文本文档中，不使用附加前导。
使用 StarSuite 6.0/7 行距
如果此选项关闭，将应用新过程以成比例的行距格式化文本行。如果此选项打开，将应用原先的方法以成比例的行距格式化文本行。
在由您当前版本的 Writer 创建的文本文档以及最新版本的 Microsoft Word 文档中，使用新过程。在由 StarSuite 8 或 OpenOffice.org 2.0 之前版本的 Writer 创建的文本文档中，使用早先的过程。
增加表格单元格底部的段落和表格间距
指定在段落底部添加的间距，即使该段落是表格单元格中的最后一段。
如果该选项关闭，表格单元格的格式化方法与 StarSuite 8 或 OpenOffice.org 2.0 之前版本的 Writer 相同。如果该选项打开，则应用另一种方法格式化表格单元格。对于由 %PRODUCTNAME %PRODUCTVERSION 创建的新文档或从 Microsoft Word 格式导入的文档，该选项默认是打开的。
使用 StarSuite 6.0/7 对象定位
指定如何计算定位到一个字符或段落的浮动对象相对于顶部和底部的段落间距的位置。
如果该选项打开，浮动对象的定位方法与 StarSuite 8 或 OpenOffice.org 2.0 之前版本的 Writer 相同。如果该选项关闭，则使用另一种方法定位浮动对象，该方法与 Microsoft Word 所使用的方法类似。
对于新文档，此选项设置为关闭。对于由 %PRODUCTNAME %PRODUCTVERSION 之前的版本创建的 Writer 文档，此选项为打开。
在对象周围使用 StarSuite 6.0/7 文字环绕
对于浮动屏幕对象周围的环绕文字，MS Word 和 Writer 分别有不同的处理方法。浮动屏幕对象是 Writer 框架和绘图对象，而在 MS Word 中是“文本框”、“图形”、“框架”、“图片”等对象。
在 MS Word 和当前版本的 Writer 中，页眉/页脚内容和脚注/尾注内容不环绕浮动屏幕对象。正文内容环绕锁定在页眉中的浮动屏幕对象。
在 StarSuite 8 或 OpenOffice.org 2.0 之前版本的 Writer 中，情况正好相反。
如果该选项关闭（默认设置），应用新的文本环绕方式。如果该选项打开，则应用先前的文本环绕方式。
定位对象时考虑环绕样式
指定定位浮动对象（定位至一个字符或段落）的复杂过程如何工作。在 StarSuite 8 或 OpenOffice.org 2.0 之前版本的 Writer 中，使用迭代过程，而在当前版本中，使用与 Microsoft Word 类似的直接过程。
如果该选项关闭，对象定位使用旧的 %PRODUCTNAME 迭代过程。如果该选项打开，则使用新的直接过程以确保与 Microsoft Word 文档的兼容。
在两端对齐的段落中用手动换行符扩展行间字词的间距
如果启用，Writer 会添加字词间的间距，这些字词位于两端对齐的段落以 Shift+Enter 组合键结尾的行中。如果禁用，将不会扩展字词间的间距以使行两端对齐。
对于 .odt 文本文档此设置默认开启。它将与文档一起以 .odt 格式保存并装入。此设置不能保存在旧的 .sxw 文本文档中，因此对于 .sxw 文本文档此设置是关闭的。

单击此按钮，将此选项卡页面上的设置作为今后 %PRODUCTNAME 会话的默认设置。

增加页首位置的段落和表格间距（当前文档中）
增加表格单元格底部的段落和表格间距
在两端对齐的段落中用手动换行符扩展行间字词的间距
