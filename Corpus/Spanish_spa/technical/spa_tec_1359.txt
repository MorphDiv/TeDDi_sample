Datos (para documentos de formulario XML)
La página de la ficha Datos del cuadro de diálogo Propiedades de un documento de formulario XML contiene opciones para formularios XML.
Datos
disponibles para un control, dependen del control correspondiente.
Sólo se mostrarán las opciones disponibles para el control y contexto actuales.
Los campos siguientes están disponibles:
Modelo de datos XML
Seleccione un modelo en la lista de modelos del documento actual.
Enlaces
Selecciona o escriba el nombre de un enlace.
Al seleccionar el nombre de un enlace se asocia con el control de formulario.
Si se escribe un nombre nuevo, se crea un enlace y se asocia con el control de formulario.
Expresión de enlace
Especifique el nodo DOM con el que enlazar el modelo de control.
Haga clic en el botón de puntos suspensivos (...) para abrir un cuadro de diálogo en el que introducir la expresión XPath.
Requerido
Especifica si el elemento debe incluirse en XForm.
Relevante
Declara el elemento como relevante.
De sólo lectura
Declara el elemento como de sólo lectura.
Restricción
Declara el elemento como restricción.
Cálculo
Declara que el elemento está calculado.
Tipo de datos
Seleccione un tipo de datos para validar el control.
x
Seleccione un tipo de datos definido por el usuario y haga clic en el botón para eliminar el tipo de datos definido por el usuario.
+
Haga clic en el botón para abrir un cuadro de diálogo en el que puede especificar el nombre de un nuevo tipo de datos definido por el usuario.
El nuevo tipo de datos hereda todas las facetas del tipo de datos seleccionado.
A continuación se enumeran todas las facetas válidas para tipos de datos.
Algunas facetas sólo están disponibles para determinados tipos de datos.
Espacios en blanco
Especifica cómo se deben tratar los espacios en blanco al procesar una cadena del tipo de datos actual.
Los valores posibles son Preserve (mantener), Replace (reemplazar) y Collapse (contraer).
La semántica de uso debe ajustarse a las definiciones de http://www.w3.org/TR/xmlschema-2/#rf-whiteSpace.
Modelo
Especifica un modelo de expresión regular.
Las cadenas que se validen con este tipo de datos deben cumplir este modelo para ser válidas.
La sintaxis de tipo de datos XSD para expresiones regulares es diferente de la que se utiliza en otras partes de %PRODUCTNAME, por ejemplo, en el cuadro de diálogo Buscar y reemplazar.
Dígitos (total)
Especifica el número máximo total de dígitos que los valores del tipo de datos decimal pueden tener.
Dígitos (fracción)
Especifica el número máximo total de dígitos fraccionales que los valores del tipo de datos decimal pueden tener.
Máx.
(inclusivo)
Especifica un límite superior inclusivo para los valores.
Máx.
(exclusivo)
Especifica un límite superior exclusivo para los valores.
Mín.
(inclusivo)
Especifica un límite inferior inclusivo para los valores.
Mín.
(exclusivo)
Especifica un límite inferior exclusivo para los valores.
Longitud
Especifica el número de caracteres de una cadena.
