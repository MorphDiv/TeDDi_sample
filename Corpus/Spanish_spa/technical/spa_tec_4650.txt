& kexi; básico
Bases de datos de & kexi;
Muchas aplicaciones, como OpenOffice. org o Microsoft Excel, crean archivos que se llaman documentos. & kexi; crea también archivos, pero aquí nos referiremos a ellos como archivos de base de datos de & kexi; o simplemente archivos de base de datos. Los archivos de base de datos de & kexi; suelen tener la extensión. kexi.
Además de almacenar las bases de datos en archivos de base de datos, & kexi; puede usar bases de datos en servidores de bases de datos, de ahí que nos refiramos a archivos de bases de datos y no simplemente a bases de datos.
El término proyecto & kexi; o, simpelemente, proyecto se usa para referirse a la base de datos de & kexi;, sin importar si está almacenada en un archivo o un servidor de bases de datos.
Crear una nueva base de datos en archivo
Inicie & kexi; o, si ya está funcionando, use Ctrl; N Archivo Nuevo.
Seleccione Base de datos vacía y pulse el botón Aceptar para confirmar la creación del proyecto.
Seleccione Proyecto nuevo almacenado en un archivo y continúe con Siguiente.
Introduzca un nombre para el proyecto y haga clic en Siguiente.
Use el navegador de archivos para elegir una carpeta en la que quiera guardar el archivo de la base de datos. Puede cambiar el nombre en la casilla Ubicación: si no le gusta la que se sugiere.

La ventana principal de & kexi;
El Navegador de proyectos y el Editor de propiedades se muestran en las zonas laterales de la ventana hija. Se pueden cambiar de tamaño u ocultar si es necesario. Se puede ocultar una zona haciendo clic en la cruz pequeña que hay en la zona superior (justo bajo la barra de herramientas).
Los objetos de base de datos (tablas, consultas, etc.) se muestran en el Navegador del proyecto, que se puede abrir haciendo clic (o doble clic, según tenga configuradas las opciones de & kde;) en sus nombres.
Elementos de la aplicación principal
Los elementos principales de la ventana de la aplicación de & kexi; son:
Barra de menú
contiene los comandos disponibles para la aplicación. Tiene una descripción detallada de todos los comandos en el apéndice.
Barra de herramientas
contiene los comandos utilizados con más frecuencia.
Panel del navegador de proyectos
contiene una lista de todos los objetos (tablas, consultas, formularios,...) que se han creado en el proyecto de la base de datos abierta. El navegador también contiene una pequeña barra de herramientas con los comandos más usados relacionados con los objetos de la base de datos.
Área de objetos de las bases de datos abiertas
un área central de la aplicación ocupa la mayor parte del espacio de la pantalla. En el modo de interfaz de usuario IDEAl, contiene solapas con las ventanas que están maximizadas. Para el modo de interfaz de usuario Childframe, contiene las ventanas flotantes.
Panel de Propiedades
contiene una lista con las propiedades del objeto de la base de datos activado. Para ciertos objetos (eg;, elementos de formularios) puede tener varias pestañas.
Barra de tareas
contiene una lista con las ventanas abiertas actualmente con objetos de base de datos. En el modo de interfaz de usuario IDEAl, está disponible como varias solapas. En el modo de interfaz Childframe, está disponible como varios botones, comportándose igual que la barra de tareas del sistema operativo.
Panel del navegador de proyectos
El panel del navegador de proyectos es uno de los elementos más utilizados de la ventana principal de & kexi;. La zona contiene una lista con todos los objetos creados en el proyecto de base de datos abierto en & kexi;. Los objetos se dividen en grupos: tablas, consultas, formularios.
El panel del navegador de proyectos también contiene una pequeña barra de herramientas para los comandos más habituales (de izquierda a derecha): Abrir objeto seleccionado, Diseñar objeto seleccionado, Crear un objeto nuevo y Eliminar objeto seleccionado.
Para cada objeto seleccionado en la lista hay disponible un menú contextual usando el & RMB;. Por ejemplo, este es el menú contextual para la tabla personas.
Al hacer doble clic con el & LMB; sobre el nombre del objeto de la lista se permite abrir el objeto en la vista de datos. Si la ventana del objeto ya estaba abierta, la acción simplemente activa la ventana sin cambiar el modo de vista.
Tenga en cuenta que su sistema operativo o su gestor de ventanas se puede configurar para gestionar clics individuales en lugar de doble clics. En ese caso, es suficiente que haga un clic sobre el nombre del objeto para abrir su ventana.

Comandos relacionados con las ventanas para objetos
Cerrar una ventana para objeto
Cuando se utiliza el modo de la interfaz de usuario IDEAl (el predefinido), cada ventana tiene su propia pestaña. Ponga el puntero del ratón sobre el icono de la pestaña. Se hace visible un botón de cierre. Pulse el botón Cerrar para cerrar la pestaña.
En el modo Childframe, a la derecha de cada ventana abierta hay botones que puede utilizar para controlar la ventana. Haga clic en el primero de la derecha para cerrar la ventana.
Además, e independientemente del modo de interfaz de usuario, puede seleccionar Ventana Cerrar en la barra de menú.
Botones de ventana para el modo de interfaz de usuario Childframe
El otro botón se pueden utilizar los demás botones (de derecha a izquierda) para: maximizar, minimizar y despegar la ventana.
Hay un icono pequeño en el lado izquierdo de la barra de título, en el que se puede hacer clic para mostrar un menú contextual con órdenes relacionadas con la ventana.
Panel del Editor de propiedades
En el panel del editor de propiedades puede cambiar las propiedades del objeto que se muestra en la ventana activa. Según el contexto, la zona está compuesta por una pestaña o por más. La primera pestaña de propiedades, que siempre está visible, contiene la lista de las propiedades disponibles.
Reglas para usar el editor de propiedades:
Cada fila contiene una sola propiedad.
Puede usar el ratón o el teclado para cambiar los valores de las propiedades particulares.
Los tipos de valores de propiedades más habituales son:
un número. Puede introducir el valor directamente o incrementar o decrementar su valor haciendo clic con el & LMB; sobre la flechas.
texto
lista desplegable de valores
Sí/ No. Puede cambiar el valor pulsando en el botón. Sí (cierto) significa que el botón está pulsado, No (falso) significa que el botón no está pulsado.
No es necesario para confirmar los valores que cambian: los cambios son visibles de forma inmediata, después de desplazarse a una fila diferente de la lista del editor de propiedades o presionando la tecla Intro.
Los nombres de las propiedades editadas recientemente que aún no estén almacenadas en la base de datos se marcan con el texto en negrita.
Después de cambiar el valor de una propiedad, aparece un botón especial Deshacer cambios aparece en el lado derecho de la lista del editor de propiedades. Al pulsar puede volver al valor original que se cargó de la base de datos al abrir el objeto. El botón sólo es visible cuando la propiedad está resaltada.
La zona del editor de propiedades aparece vacía si:
no hay abierto ningún objeto individual de la base de datos, o
la ventana de objeto de la base de datos activa no ofrece propiedades. Suele ser así cuando se abre la vista de datos en lugar de la vista de diseño

Notas
De forma predefinida, la lista desplegable Filtro: tiene seleccionada Proyecto de base de datos de Kexi basado en archivo. En caso de que el archivo que esté buscando tenga otra extensión, puede cambiar la selección de la lista desplegable Filtro: para que sea Todos los archivos y se muestren todos los archivos disponibles (sin tener en cuenta la extensión).
Si ha seleccionado un archivo de un tipo externo como un archivo. mdb de MS Access, & kexi; le brindará la opción de importar el archivo.
Si ha seleccionado un archivo de datos de conexión (con extensión. kexic) o un archivo de acceso rápido a un proyecto en un servidor de base de datos (con extensión. kexis), & kexi; mostrará el diálogo adecuado.
Abrir un archivo de base de datos de & kexi; pulsando en el icono del archivo. kexi
Pulse en el icono del archivo usando el gestor de archivos o el escritorio. & kexi; abre automáticamente el proyecto de base de datos del archivo.
Aviso sobre acceso remoto a archivos de bases de datos. Puede querer abrir un archivo de base de datos que esté ubicado en un lugar remoto (eg;, una web o un servidor FTP o un recurso compartido de MS Windows). K Desktop Environment le permite abrir archivos de lugares remotos directamente en las aplicaciones y para guardar los cambios en el mismo lugar, una copia del archivo se descarga a un directorio temporal de su computador y todos los cambios que haya hecho al archivo local. El original, remoto, del archivo permanece sin cambios, por lo que se recomienda que copie (descargue) el archivo en su computador, abra el archivo y lo copie en el origen remoto si quiere actualizarlo.
