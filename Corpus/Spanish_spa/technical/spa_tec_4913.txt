cheatmake
1
engaña a make para que no reconstruya ciertos archivos.
cheatmake hidechange archivo show why archivo
Descripción
El comando cheatmake se usa para ahorrar tiempo durante las recompilaciones. Puede engañar a Make para que pase por alto los archivos cuyas modificaciones no tengan importancia como por ejemplo cuando añade comentarios en el código.
Esta herramienta forma parte del kit de desarrollo de & kde;.
Opciones
hidechange archivo
Oculta el hecho de que el archivo haya cambiado poniendo la marca de tiempo en un momento anterior. Úselo con precaución.
Mostrar
Lista que archivos que make necesita reconstruir.
why archivos
Explica la razón por la cual make necesita reconstruir un determinado archivo
Entorno
Una de las siguientes variables (pero no ambas) debería configurarse si el directorio donde se compila es distinto del que contiene el código fuente. Si el directorio de compilación es un subdirectorio dentro del código fuente, debería usar la variable OBJ_SUBDIR.
OBJ_SUBJDIR
Indica que el directorio de compilación está en un subdirectorio dentro del código fuente. Por ejemplo, si el directorio del código fuente es kdesdk y el de compilación es kdesdk/ obj-i386-linux entonces OBJ_SUBDIR debería tener el valor de obj-i386-linux.
OBJ_REPLACEMENT
Para transformar el directorio del código fuente en el de compilación se utiliza una expresión de sed. Por ejemplo si el directorio de los fuentes es kdesdk/ obj-i386-linux, entonces OBJ_REPLACEMENT debería valer s#kdesdk#kdesdk-obj#.
