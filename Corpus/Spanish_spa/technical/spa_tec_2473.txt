Colaborando
¿Cómo puedo colaborar con KDE?
KDE es un proyecto de software libre que sobrevive gracias a contribuciones voluntarias.
Todos pueden contribuir con KDE.
No sólo los programadores son bienvenidos.
Hay muchas maneras en las que usted puede ayudar a mejorar KDE:
Pruebe el software.
Envíe informes de fallos.
Para más información, vea ¿Cómo puedo enviar un informe de un fallo?.
Escriba documentación o archivos de ayuda.
Traduzca programas, documentación y archivos de ayuda.
Dibuje íconos atractivos o componga efectos de sonido.
Escriba artículos y libros sobre KDE.
Si usted desea ayudar a difundir la palabra de KDE, simplemente envíe un mensaje a kde-pr@kde.org.
Esto lo pondrá en contacto con los voluntarios de relaciones públicas de KDE.
Escriba nuevos programas para KDE.
Por favor remítase a para más información.
Por supuesto, los mecenas también son bienvenidos.: -)
Hay varios lugares en los que buscar más información si desea verse involucrado en el desarrollo.
El primer paso consiste en suscribirse a algunas de las lista de correo.
Usted pronto verá algo que pueda ser mejorado o añadido.
¿Cómo puedo informar de un fallo?
Hay un sistema automático de seguimiento de fallos disponible en http: / /buglist.kde.org.
El sistema incluye un formulario para enviar descripciones de nuevos fallos, así como una lista con todos los fallos conocidos.
Esto abre un cuadro dediálogo donde usted puede introducir toda la información relevante.
Por favor asegúrese de seguir todas las instrucciones del asistente de informe de fallos.
Deseo programar para KDE. ¿Qué debo hacer primero?
Todos pueden desarrollar software para el KDE.
Lo primero que usted debe hacer depende principalmente de su nivel de experiencia, por ejemplo, si conoce el lenguaje C+ + o tiene experiencia con Qt, etcétera.
Para programar para KDE, necesitará algunas herramientas básicas: automake, autoconf, y egcs.
Puede echar un vistazo en http: / /developer.kde.org / para más información.
Otra excelente fuente para aprender a programar para KDE son los tutoriales de Qt. éstos vienen instalados junto con Qt.
Los tutoriales se pueden encontrar en la sección "Usando Qt".
Por favor lea la sección Cómo puedo suscribirme a estas listas o cancelar mi suscripción cuidadosamente.
Todo lo que dice ahí se aplica también a la lista de correo para desarrolladores.
¿Cómo puedo acceder al CVS?
El proyecto KDE utiliza CVS para desarrollar las partes centrales del software.
Usualmente, cuando una parte del código ha sido cambiada (por ejemplo, al arreglar un fallo), y el desarrollador desea incluir este cambio, la mejor manera de hacerlo es crear un parche con respecto a una instantánea actual del desarrollo del código y enviarlo al desarrollador del programa o al encargado de su mantención.
Si usted está haciendo esto de forma más o menos regular, puede pedirle a Stephan Kulow, coolo@kde.org acceso directo al servidor CVS.
Hay que tener en cuenta, no obstante, que una mayor cantidad de usuarios hace más lento el acceso al servidor CVS para todos los desarrolladores, por lo que queremos mantener un número razonablemente bajo de personas con acceso directo a CVS. ¡Sin embargo, solicítelo!
Sólo tengo curiosidad. ¿Puedo tener acceso sólo de lectura al servidor CVS?
Sí.
Es posible obtener acceso sólo de lectura al depósito CVS con las utilidades cvsup o cvs.
Además, el servicio de CVS anónimo también está disponible.
Para averiguar más sobre como fijar y usar el CVS anónimo, por favor consulte aquí.
¿Hay alguna réplica de cvs para KDE?
Sí.
El servidor principal de KDE normalmente está muy ocupado.
Por favor vaya a http: / /www.kde.org / cvsupmirrors.html y http: / /www.kde.org / anoncvs.html para ver la lista de réplicas disponibles.
Una vez haya escogido un nuevo servidor de cvsup, simplemente reemplace *default host=cvs.kde.org por *default host=cvsup.su.nuevo.servidor en su script de cvsup.
Las instrucciones para cambiar de servidor anoncvs se encuentran en http: / /www.kde.org / anoncvs.html, también.
