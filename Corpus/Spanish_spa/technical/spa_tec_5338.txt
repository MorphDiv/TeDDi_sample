Resumen
meinproc4 --help Generic-options Qt-options KDE-options --cache file --check --htdig --srcdir --param key = value -o --output file --stdout --stylesheet xsl docbook_file
Descripción
meinproc4 convierte archivos DocBook a HTML.
Opciones genéricas
--author
Muestra información sobre el autor.
--help
Muestra ayuda sobre las opciones.
--help-all
Muestra todas las opciones.
--help-kde
Muestra las opciones específicas de KDE.
--help-qt
Muestra las opciones especificas de QT.
--license
Muestra información sobre la licencia.
-v --version
Muestra información de la versión
Opciones de la aplicación
--cache file
Crea un archivo de caché para el documento
--check
Verifica la validez de la documentación
La salida puede ser un poco desalentadora, ya que un pequeño error puede causar una cascada de errores. El truco es buscar el primer error, corregirlo, guardar el archivo y ejecutar meinproc4 de nuevo.
--htdig
Crea un índice compatible con ht: / /dig
-o, --output file
Vuelca el documento completo a file.
--stdout
Vuelca el documento completo a stdout.
--stylesheet xsl
Hoja de estilos a emplear
--srcdir dir
Ajusta el directorio raíz para la búsqueda de kdelibs
--param key=value
Ajusta los parámetros que se pasan a la hoja de estilo.
Utilización
El modo mas habitual de ejecutar meinproc4 es sencillamente como meinproc4 docbook-file donde docbook-file es generalmente index. docbook. Esta orden genera páginas HTML a partir del archivo DocBook. Estas páginas sólo son visibles an navegadores basados en KDE (como Konqueror).
Si necesita ver la salida HTML en otro navegador (por ejemplo, si la está haciendo disponible on line), utilice meinproc4 --stylesheet stylesheet-name docbook-file donde stylesheet-name es la ruta completa a una de las hojas de estilo XSL en $KDEDIR/ share/ apps/ ksgmltools/ customization. Para generar salida adecuada para la web, puede emplear kde-web. xsl or kde-chunk-online. xsl. Vea el archivo README en este directorio para mas detalles.
