Introducción
¿Qué es KDE?
KDE es el Entorno de Escritorio K.
Es un proyecto iniciado por Matthias Ettrich en 1996.
El objetivo del proyecto KDE es combinar el poder de los sistemas operativos UNIX con la comodidad de una interfaz de usuario moderna.
¡En resumen, KDE llevará a UNIX al escritorio!
¿En qué plataformas puedo esperar que KDE funcione?
KDE es un entorno de escritorio para todas las variantes de UNIX.
Si bien es cierto que la mayoría de los desarrolladores de KDE usan Linux, KDE funciona bien en una amplia gama de sistemas.
Puede,sin embargo, ser necesario modificar un poco el código fuente para conseguir que KDE se compile para una versión no tan popular de UNIX ,o bien si no se utilizan las herramientas de desarrollo GNU, en particular el compilador gcc.
Para una lista casi completa de sistemas en los que funciona KDE por favor consulte la lista de sistemas para KDE
¿Es KDE un administrador de ventanas?
No, KDE no es un administrador de ventanas.
Aunque KDE incluye un administrador de ventanas muy sofisticado (KWin), KDE es mucho más que eso.
Es un completo Entorno de Escritorio Integrado.
KDE provee un entorno de escritorio completo, incluyendo un administrador de archivos, un administrador de ventanas, un sistema de ayuda, un sistema de configuración, innumerables herramientas y utilidades y un número cada vez mayor de aplicaciones, incluyendo clientes de correo y noticias, programas de dibujo, un visor de PostScript y otro de DVI, y cosas por el estilo.
¿Es KDE un clon de CDE, Windows 95 o el Mac?
No, KDE no es un clon.
Específicamente, KDE no es un clon de CDE o Windows.
Aunque los desarrolladores de KDE han sacado (y continuarán haciéndolo) las mejores características de todos los entornos de escritorio, KDE es un entorno realmente único que sigue y continuará siguiendo su propio camino.
¿Es KDE software libre?
Sí, KDE es software libre de acuerdo a la Licencia Pública General GNU.
Todas las librerías de KDE están disponibles bajo la licencia LGPL haciendo posible el desarrollo de software comercial para KDE, pero todos los programas de KDE están bajo la licencia GPL.
KDE utiliza el conjunto de herramientas multiplataforma Qt, escrito en C++, que también está disponible (desde la versión 2.2) bajo la licencia GPL.
Es completamente legal distribuir el KDE y Qt en CD-ROM, sin costo alguno.
No se incurre en ninguna clase de cuotas u honorarios.

El método más cómodo para iniciar KDE es mediante el script startkde.
Simplemente ponga la línea startkde al final de su archivo .xsession (o su archivo .xinitrc o .Xclients si no está usando kdm o xdm).

Si no hay un archivo .xsession, .xinitrc, o .Xclients en su directorio personal, simplemente cree uno nuevo con una sola línea que diga startkde.

En la consola aparecen errores relacionados con DCOPserver. ¿Qué está ocurriendo?
KDE utiliza varios archivos temporales durante su funcionamiento.
Normalmente se almacenan en las siguientes ubicaciones:
~ / .DCOPserver-* (normalmente hay dos de estos, uno de ellos es un enlace al otro)
~ / .kde / socket - nombre_del_equipo
/tmp / tmp-kde - USUARIO
/tmp / ksocket - USUARIO
Si lo enlaces simbólico se rompen, normalmente porque un cron o un guión de apagado vacian el directorio / tmp, pueden ocurrir cosas extrañas.
Si lo único que obtiene al iniciar KDE es una pantalla gris o un mensaje diciendo Compruebe su instalación, cierre el entorno X y elimine todos los archivos indicados anteriormente, depués vuelva a intentar arrancar el servidor X.
Normalmente (es decir cuando no se actualiza entre versiones de KDE) es bastante seguro dejar esos archivos intactos, así se puede ahorrar unos segundos al iniciar KDE.

Al iconizar una ventana, desaparece. ¿A dónde va?
En el caso de muchas interfaces gráficas para X Window System, el botón para minimizar (el que tiene el pequeño punto) borra la ventana y crea, en su lugar, un ícono en el escritorio.
KDE no hace esto.
En vez de eso, cuando una ventana es iconizada, simplemente es ocultada (pero el programa continúa funcionando).
Hay varias maneras de acceder a ventanas "desaparecidas":
Si usted está ejecutando la barra de tareas de Kicker, puede optar por tener una lista de tareas en su escritorio.
Las tareas iconizadas tendrán paréntesis rodeando sus nombres.
Si hace click con el botón medio del ratón en la ventana raíz (es decir, el fondo del escritorio), KWin mostrará una lista de todas las tareas disponibles (y, una vez más, los nombres de las tareas iconizadas estarán rodeados por paréntesis).
¿Cómo maximizo ventanas sólo vertical u horizontalmente?
Si su ventana todavía no ha sido maximizada, hacer click en el botón de maximización con el botón izquierdo, medio o derecho del ratón maximizará la ventana completamente, en orientación vertical y en orientación horizontal respectivamente.
¿Qué es "recoger" (shading) una ventana?
"Recoger" una ventana significa recoger una ventana dejando sólo la barra de título visible.
Esto se puede hacer haciendo doble click en la barra de título de la ventana.

Preguntas variadas
¿Por qué KDE hace uso de Qt?
Qt es un conjunto muy sofisticado de herramientas que proveen de todo lo necesario para construir una interfaz de usuario moderna.
Qt está escrito en C++, permitiendo así el desarrollo orientado a objetos, que asegura la eficiencia y la reutilización del código en un proyecto del tamaño y la envergadura de KDE.
En nuestra opinión, no existe un mejor conjunto de herramientas disponible para sistemas UNIX, y habría sido un grave error haber construído KDE sobre una base que no fuese lamejor.
¿Por qué KDE no hace uso de gtk, xforms, xlib, etcétera?
Hay muchos conjuntos de herramientas disponibles.
Para ofrecer una interfaz de usuario de usuario consistente y utilizar un mínimo de recursos, como la memoria, KDE sólo puede usar uno de ellos.
Qt fue elegido por las razones antes mencionadas.
Pero Qt no es software libre, ¿o sí?
¡Sí que lo es!
Desde el 4 de septiembre del 2000, la versión 2.2 de las librerías Qt está sujeta a la licencia GPL ,cumpliendo así con todos los requisitos del software libre.

¿Qué es CVS?
Son las siglas de Concurrent Versions System (Sistema de Versiones Concurrentes).
Es un sistema de control de versiones y está basado en RCS (Revision Control System, Sistema de Control de Revisiones), pero ofrece mayor funcionalidad.
Seutiliza para mantener código fuente en desarrollo.
Permite mantener múltiples versiones de diversos archivos (práctico si algo se echa a perder y se debe usar una copia de seguridad más antigua), y permite a personas en distintos puntos de la Red obtener las últimas versiones del código fuente e incluso enviar archivos nuevos si tienen permiso para ello.
También es software de código abierto (se puede pagar para obtener soporte si es preciso), y ya que es software libre, es el sistema de elección para las personas que escriben software gratuito, como KDE.

¿Cómo puedo acceder al CVS?
El proyecto KDE utiliza CVS para desarrollar las partes centrales del software.
Usualmente, cuando una parte del código ha sido cambiada (por ejemplo, al arreglar un fallo), y el desarrollador desea incluir este cambio, la mejor manera de hacerlo es crear un parche con respecto a una instantánea actual del desarrollo del código y enviarlo al desarrollador del programa o al encargado de su mantención.
Si usted está haciendo esto de forma más o menos regular, puede pedirle a Stephan Kulow, coolo@kde.org acceso directo al servidor CVS.
Hay que tener en cuenta, no obstante, que una mayor cantidad de usuarios hace más lento el acceso al servidor CVS para todos los desarrolladores, por lo que queremos mantener un número razonablemente bajo de personas con acceso directo a CVS. ¡Sin embargo, solicítelo!
